## 一、核心概念

### 1.1 View 动画 vs 属性动画

| 维度 | View 动画（补间动画） | 属性动画 |
|------|----------------------|---------|
| 包名 | `android.view.animation` | `android.animation` |
| 作用对象 | 仅 View | 任意对象的任意属性 |
| 是否改变属性 | ❌ 只改变绘制位置，不改变实际属性 | ✅ 真正修改对象属性值 |
| 点击事件 | 动画后点击区域仍在原位 | 跟随动画移动 |
| 支持类型 | 平移/缩放/旋转/透明度 | 任意可 get/set 的属性 |
| API 级别 | API 1 | API 11+ |

> ⚠️ View 动画的"假动画"问题是面试高频考点：动画结束后 View 的 `getX()/getY()` 不变，点击事件响应区域也不变。

### 1.2 属性动画三大核心类

#### ValueAnimator — 值动画器

属性动画的基石，负责计算动画过程中每一帧的值，但**不直接操作对象属性**。

```java
ValueAnimator animator = ValueAnimator.ofFloat(0f, 1f);
animator.setDuration(300);
animator.addUpdateListener(a -> {
    float value = (float) a.getAnimatedValue();
    view.setAlpha(value);  // 手动设置属性
});
animator.start();
```

#### ObjectAnimator — 对象动画器

继承自 [[ValueAnimator]]，内部通过**反射或 Property 对象**自动调用目标对象的 setter。

```java
// 方式一：字符串属性名（反射）
ObjectAnimator anim1 = ObjectAnimator.ofFloat(view, "translationX", 0f, 200f);
anim1.setDuration(500);
anim1.start();

// 方式二：Property 对象（无反射，推荐）
ObjectAnimator anim2 = ObjectAnimator.ofFloat(view, View.TRANSLATION_X, 0f, 200f);
anim2.setDuration(500);
anim2.start();
```

> `ObjectAnimator` 要求目标属性必须有对应的 `setXxx()` 方法；若还需要自动获取初始值，则还需 `getXxx()`。

#### AnimatorSet — 动画集合

编排多个 [[Animator]] 的播放顺序：

```java
ObjectAnimator scaleX = ObjectAnimator.ofFloat(view, View.SCALE_X, 1f, 1.5f);
ObjectAnimator scaleY = ObjectAnimator.ofFloat(view, View.SCALE_Y, 1f, 1.5f);
ObjectAnimator alpha  = ObjectAnimator.ofFloat(view, View.ALPHA, 1f, 0.5f);

AnimatorSet set = new AnimatorSet();
// 缩放同时执行，之后执行透明度
set.play(scaleX).with(scaleY);
set.play(alpha).after(scaleX);
set.setDuration(400);
set.start();
```

### 1.3 TimeInterpolator 与 TypeEvaluator

属性动画的两大数学引擎：

```
时间分数 (0→1)
    │
    ▼
TimeInterpolator  ──→  插值后的分数 (可 >1 或 <0)
                            │
                            ▼
                    TypeEvaluator  ──→  当前帧的属性值
```

- **TimeInterpolator（插值器）**：决定动画的**速度曲线**，输入为时间比例 `t ∈ [0,1]`，输出为进度比例。
- **TypeEvaluator（估值器）**：根据插值器输出的进度，计算**具体属性值**（如颜色、坐标、路径点等）。

系统内置插值器：

| 插值器 | 效果 |
|--------|------|
| `LinearInterpolator` | 匀速 |
| `AccelerateInterpolator` | 加速 |
| `DecelerateInterpolator` | 减速 |
| `AccelerateDecelerateInterpolator` | 先加速后减速 |
| `OvershootInterpolator` | 超出终点后回弹 |
| `AnticipateInterpolator` | 先回退再前进 |
| `BounceInterpolator` | 弹跳效果 |
| `PathInterpolator` | 贝塞尔曲线自定义 |

系统内置估值器：

| 估值器 | 用途 |
|--------|------|
| `IntEvaluator` | 整型属性 |
| `FloatEvaluator` | 浮点属性 |
| `ArgbEvaluator` | 颜色渐变 |

## 二、原理与源码

### 2.1 属性动画驱动机制：Choreographer + VSYNC

属性动画并非使用 `Handler.postDelayed` 驱动，而是依赖 [[Choreographer]] 的 VSYNC 信号：

```
┌─────────────┐
│  start()    │
└──────┬──────┘
       ▼
AnimationHandler.addAnimationFrameCallback()
       │
       ▼
Choreographer.postFrameCallback(CALLBACK_ANIMATION)
       │
       ▼
  VSYNC 信号到达 (~16.6ms / 帧)
       │
       ▼
AnimationHandler.doAnimationFrame()
       │
       ▼
ValueAnimator.doAnimationFrame(frameTime)
       │
       ▼
animateValue(fraction) → 调用 Interpolator + Evaluator → 通知 UpdateListener
       │
       ▼
再次 postFrameCallback → 循环直到动画结束
```

### 2.2 AnimationHandler 源码关键逻辑

`AnimationHandler` 是一个**线程单例**（ThreadLocal），管理当前线程所有活跃动画：

```java
// AnimationHandler.java 核心简化
class AnimationHandler {
    // ThreadLocal 保证线程隔离
    private static final ThreadLocal<AnimationHandler> sAnimatorHandler = new ThreadLocal<>();

    private final ArrayList<AnimationFrameCallback> mAnimationCallbacks = new ArrayList<>();

    // Choreographer 回调
    private final Choreographer.FrameCallback mFrameCallback = frameTimeNanos -> {
        doAnimationFrame(getProvider().getFrameTime());
        if (mAnimationCallbacks.size() > 0) {
            // 还有动画未结束，继续注册下一帧回调
            getProvider().postFrameCallback(this.mFrameCallback);
        }
    };

    void addAnimationFrameCallback(AnimationFrameCallback callback) {
        if (mAnimationCallbacks.size() == 0) {
            // 第一个动画注册时，开始监听 VSYNC
            getProvider().postFrameCallback(mFrameCallback);
        }
        mAnimationCallbacks.add(callback);
    }

    private void doAnimationFrame(long frameTime) {
        for (int i = 0; i < mAnimationCallbacks.size(); i++) {
            final AnimationFrameCallback callback = mAnimationCallbacks.get(i);
            callback.doAnimationFrame(frameTime);
        }
    }
}
```

> 关键点：动画必须在有 [[Looper]] 的线程启动，因为 Choreographer 依赖 Looper 接收 VSYNC。

### 2.3 ValueAnimator.animateValue() 核心流程

```java
// ValueAnimator.java 简化
void animateValue(float fraction) {
    // 1. 插值器：时间分数 → 动画进度
    fraction = mInterpolator.getInterpolation(fraction);
    mCurrentFraction = fraction;

    // 2. 估值器：动画进度 → 具体属性值
    for (int i = 0; i < numValues; i++) {
        mValues[i].calculateValue(fraction);
    }

    // 3. 通知监听器
    for (int i = 0; i < numListeners; i++) {
        mUpdateListeners.get(i).onAnimationUpdate(this);
    }
}
```

### 2.4 常用插值器数学公式

#### LinearInterpolator

$$f(t) = t$$

#### AccelerateInterpolator（factor = 1.0）

$$f(t) = t^2$$

#### DecelerateInterpolator（factor = 1.0）

$$f(t) = 1 - (1 - t)^2$$

#### AccelerateDecelerateInterpolator

$$f(t) = \frac{\cos((t + 1) \cdot \pi)}{2} + 0.5$$

#### OvershootInterpolator（tension = 2.0）

$$f(t) = (t - 1)^2 \cdot ((tension + 1) \cdot (t - 1) + tension) + 1$$

#### BounceInterpolator（分段函数）

```java
// 源码等价 Java 实现
float bounce(float t) {
    if (t < 1.0 / 2.75) {
        return 7.5625f * t * t;
    } else if (t < 2.0 / 2.75) {
        t -= 1.5f / 2.75f;
        return 7.5625f * t * t + 0.75f;
    } else if (t < 2.5 / 2.75) {
        t -= 2.25f / 2.75f;
        return 7.5625f * t * t + 0.9375f;
    } else {
        t -= 2.625f / 2.75f;
        return 7.5625f * t * t + 0.984375f;
    }
}
```

## 三、高级用法

### 3.1 自定义插值器

实现 `TimeInterpolator` 接口，重写 `getInterpolation(input: Float): Float`。

示例：弹性插值器（Spring 效果）

```java
public class SpringInterpolator implements TimeInterpolator {
    private final float factor;

    public SpringInterpolator() { this(0.4f); }
    public SpringInterpolator(float factor) { this.factor = factor; }

    @Override
    public float getInterpolation(float input) {
        // 衰减正弦波：模拟弹簧振荡
        return (float) (Math.pow(2.0, -10.0 * input) *
            Math.sin((input - factor / 4) * (2 * Math.PI) / factor) + 1);
    }
}

// 使用
ObjectAnimator anim = ObjectAnimator.ofFloat(view, View.TRANSLATION_Y, 0f, -300f);
anim.setInterpolator(new SpringInterpolator());
anim.setDuration(800);
anim.start();
```

> 💡 也可以用 `PathInterpolator` 通过贝塞尔曲线实现自定义速度曲线，无需写数学公式：

```java
// 三阶贝塞尔：(0,0) → (cx1,cy1) → (cx2,cy2) → (1,1)
PathInterpolator pathInterpolator = new PathInterpolator(0.25f, 0.1f, 0.25f, 1.0f);
```

### 3.2 自定义估值器

实现 `TypeEvaluator<T>` 接口，用于计算**非标准类型**的中间值。

示例：PointF 估值器（沿贝塞尔曲线运动）

```java
public class BezierPoint {
    public final float x, y;
    public BezierPoint(float x, float y) { this.x = x; this.y = y; }
}

public class BezierEvaluator implements TypeEvaluator<BezierPoint> {
    private final BezierPoint control;  // 控制点
    public BezierEvaluator(BezierPoint control) { this.control = control; }

    @Override
    public BezierPoint evaluate(float t, BezierPoint start, BezierPoint end) {
        // 二阶贝塞尔公式: B(t) = (1-t)²P0 + 2t(1-t)P1 + t²P2
        float oneMinusT = 1 - t;
        return new BezierPoint(
            oneMinusT * oneMinusT * start.x + 2 * oneMinusT * t * control.x + t * t * end.x,
            oneMinusT * oneMinusT * start.y + 2 * oneMinusT * t * control.y + t * t * end.y
        );
    }
}

// 使用：模拟抛物线轨迹
ValueAnimator animator = ValueAnimator.ofObject(
    new BezierEvaluator(new BezierPoint(300f, -200f)),
    new BezierPoint(0f, 0f),
    new BezierPoint(600f, 600f)
);
animator.setDuration(1000);
animator.addUpdateListener(it -> {
    BezierPoint point = (BezierPoint) it.getAnimatedValue();
    view.setTranslationX(point.x);
    view.setTranslationY(point.y);
});
animator.start();
```

示例：HSV 颜色估值器（比 ArgbEvaluator 过渡更自然）

```java
public class HsvEvaluator implements TypeEvaluator<Integer> {
    private final float[] startHsv = new float[3];
    private final float[] endHsv = new float[3];

    @Override
    public Integer evaluate(float fraction, Integer startColor, Integer endColor) {
        Color.colorToHSV(startColor, startHsv);
        Color.colorToHSV(endColor, endHsv);
        float h = startHsv[0] + (endHsv[0] - startHsv[0]) * fraction;
        float s = startHsv[1] + (endHsv[1] - startHsv[1]) * fraction;
        float v = startHsv[2] + (endHsv[2] - startHsv[2]) * fraction;
        return Color.HSVToColor(new float[]{h, s, v});
    }
}
```

### 3.4 MotionLayout 简介

[[MotionLayout]] 是 `ConstraintLayout` 的子类，通过 `MotionScene` XML 声明式定义复杂动画：

```xml
<!-- activity_main.xml -->
<androidx.constraintlayout.motion.widget.MotionLayout
    android:id="@+id/motionLayout"
    app:layoutDescription="@xml/scene_main"
    ... >

    <ImageView android:id="@+id/image" ... />
</androidx.constraintlayout.motion.widget.MotionLayout>
```

```xml
<!-- xml/scene_main.xml -->
<MotionScene>
    <Transition
        app:constraintSetStart="@id/start"
        app:constraintSetEnd="@id/end"
        app:duration="600">
        <OnSwipe
            app:touchAnchorId="@id/image"
            app:dragDirection="dragRight" />
        <KeyFrameSet>
            <KeyPosition
                app:motionTarget="@id/image"
                app:framePosition="50"
                app:keyPositionType="pathRelative"
                app:percentY="-0.2" />
        </KeyFrameSet>
    </Transition>

    <ConstraintSet android:id="@+id/start">
        <Constraint android:id="@id/image"
            android:layout_width="60dp"
            android:layout_height="60dp"
            app:layout_constraintStart_toStartOf="parent"
            app:layout_constraintTop_toTopOf="parent" />
    </ConstraintSet>

    <ConstraintSet android:id="@+id/end">
        <Constraint android:id="@id/image"
            android:layout_width="120dp"
            android:layout_height="120dp"
            app:layout_constraintEnd_toEndOf="parent"
            app:layout_constraintBottom_toBottomOf="parent" />
    </ConstraintSet>
</MotionScene>
```

MotionLayout 优势：
- 声明式，动画逻辑与业务代码分离
- 支持手势驱动（拖拽进度 0→1）
- 支持 `KeyFrame`（关键帧）实现非线性路径
- Android Studio Motion Editor 可视化编辑

## 四、面试题

### Q1：View 动画和属性动画的本质区别是什么？为什么 View 动画后点击区域不变？

**A**：View 动画只是在 `draw()` 时对 Canvas 做矩阵变换（平移/缩放/旋转），**不修改 View 的 `mLeft/mTop/mRight/mBottom` 等布局属性**，所以 `getX()/getY()` 不变，触摸事件分发时 `ViewGroup.dispatchTouchEvent()` 仍按原始位置做 hit test。属性动画通过反射或 Property 对象**真正修改**目标属性值（如 `translationX`），View 的位置信息同步更新。

### Q2：ObjectAnimator 内部是如何找到并调用 setter 的？

**A**：
1. 如果传入的是 `Property<T, V>` 对象（如 `View.TRANSLATION_X`），直接调用 `property.set(target, value)`，无反射。
2. 如果传入的是字符串属性名 `"translationX"`，`PropertyValuesHolder` 内部通过 `target.getClass().getMethod("setTranslationX", float.class)` 反射获取 setter 并缓存。
3. 若找不到 setter，动画静默失败（不崩溃但无效果），这是常见的"动画不生效"原因。

### Q3：属性动画是如何保证与屏幕刷新同步的？

**A**：属性动画通过 [[Choreographer]] 注册 `CALLBACK_ANIMATION` 类型的帧回调。Choreographer 在收到底层 VSYNC 信号后，按 `INPUT → ANIMATION → TRAVERSAL` 顺序依次执行回调。因此属性动画的每一帧计算都与屏幕刷新对齐，避免了 `postDelayed` 方案的时间漂移和丢帧问题。

### Q4：插值器和估值器的职责分别是什么？能否只用其中一个？

**A**：
- **插值器**：将线性时间进度 `t ∈ [0,1]` 映射为非线性的动画进度（控制"快慢节奏"）。
- **估值器**：根据动画进度计算当前帧的具体属性值（控制"值怎么算"）。

两者是**流水线关系**：`时间 → 插值器 → 进度 → 估值器 → 值`。对于 `ofFloat/ofInt`，系统有默认估值器，可以只设置插值器；但对于自定义类型（如 `PointF`、颜色），必须提供自定义估值器。

### Q5：AnimatorSet 的 playTogether 和 playSequentially 底层有什么区别？

**A**：
- `playTogether()`：所有子动画同时 `start()`，各自独立运行，AnimatorSet 的总时长 = 最长子动画时长。
- `playSequentially()`：内部为每个子动画设置 `startDelay`，使其依次执行。本质上所有子动画仍然在同一帧注册，只是通过延迟错开。
- 注意：对 AnimatorSet 设置 `duration` 会**覆盖**所有子动画的 duration。

### Q6：为什么属性动画可能导致内存泄漏？如何避免？

**A**：属性动画（尤其是无限循环动画 `repeatCount = INFINITE`）持有目标 View 的引用，而 View 持有 Activity 的 Context。如果 Activity 销毁时动画未取消，引用链 `AnimationHandler → ValueAnimator → target View → Activity` 导致 Activity 无法被 GC。解决方案：
1. 在 `onDestroy()` 或 `onDetachedFromWindow()` 中调用 `animator.cancel()`
2. 使用 `Lifecycle` 感知自动取消（见第五节）

## 五、实战与踩坑

### 5.1 动画内存泄漏

最常见的泄漏场景：**无限循环动画 + Activity 销毁**。

```java
// ❌ 错误：Activity 退出后动画仍在运行，持有 View → Activity 引用
public class LeakyActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        ObjectAnimator anim = ObjectAnimator.ofFloat(binding.icon, View.ROTATION, 0f, 360f);
        anim.setRepeatCount(ValueAnimator.INFINITE);
        anim.setDuration(2000);
        anim.start();
    }
}
```

```java
// ✅ 正确：生命周期感知，自动取消
public class SafeActivity extends AppCompatActivity {
    private ObjectAnimator rotateAnimator;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        rotateAnimator = ObjectAnimator.ofFloat(binding.icon, View.ROTATION, 0f, 360f);
        rotateAnimator.setRepeatCount(ValueAnimator.INFINITE);
        rotateAnimator.setDuration(2000);

        getLifecycle().addObserver(new DefaultLifecycleObserver() {
            @Override public void onResume(@NonNull LifecycleOwner owner) { if (rotateAnimator != null) rotateAnimator.start(); }
            @Override public void onPause(@NonNull LifecycleOwner owner) { if (rotateAnimator != null) rotateAnimator.pause(); }
            @Override public void onDestroy(@NonNull LifecycleOwner owner) {
                if (rotateAnimator != null) rotateAnimator.cancel();
                rotateAnimator = null;
            }
        });
    }
}
```

### 5.2 属性动画在 onDetachedFromWindow 时取消

自定义 View 内部启动的动画，应在 `onDetachedFromWindow()` 中取消：

```java
public class PulseView extends View {

    private final ObjectAnimator pulseAnimator;

    public PulseView(Context context) { this(context, null); }
    public PulseView(Context context, AttributeSet attrs) {
        super(context, attrs);
        pulseAnimator = ObjectAnimator.ofFloat(this, SCALE_X, 1f, 1.2f);
        pulseAnimator.setRepeatMode(ValueAnimator.REVERSE);
        pulseAnimator.setRepeatCount(ValueAnimator.INFINITE);
        pulseAnimator.setDuration(600);
    }

    @Override
    protected void onAttachedToWindow() {
        super.onAttachedToWindow();
        pulseAnimator.start();
    }

    @Override
    protected void onDetachedFromWindow() {
        // ✅ 必须取消，否则 AnimationHandler 持有 this → 泄漏
        pulseAnimator.cancel();
        super.onDetachedFromWindow();
    }
}
```

> 为什么选 `onDetachedFromWindow` 而非 `onDestroy`？因为自定义 View 没有 `onDestroy` 回调，`onDetachedFromWindow` 是 View 从窗口移除时的最后通知，此时应释放所有资源。

### 5.3 帧动画 OOM

[[AnimationDrawable]]（帧动画）会将所有帧图片**一次性加载到内存**，帧数多或图片大时极易 OOM：

```xml
<!-- ❌ 危险：30 帧 × 每帧 500KB ≈ 15MB 内存 -->
<animation-list android:oneshot="false">
    <item android:drawable="@drawable/frame_01" android:duration="33"/>
    <item android:drawable="@drawable/frame_02" android:duration="33"/>
    <!-- ... 30 帧 -->
</animation-list>
```

替代方案：

```java
// ✅ 方案一：用属性动画 + ImageView 手动切帧（按需加载）
public class FrameAnimator {
    private final ImageView imageView;
    private final List<Integer> frames;  // drawable res ids
    private final long frameDuration;
    private ValueAnimator animator;

    public FrameAnimator(ImageView imageView, List<Integer> frames, long frameDuration) {
        this.imageView = imageView;
        this.frames = frames;
        this.frameDuration = frameDuration;
    }

    public FrameAnimator(ImageView imageView, List<Integer> frames) {
        this(imageView, frames, 33L);
    }

    public void start() {
        animator = ValueAnimator.ofInt(0, frames.size() - 1);
        animator.setDuration(frameDuration * frames.size());
        animator.setInterpolator(new LinearInterpolator());
        animator.addUpdateListener(it -> {
            int index = (int) it.getAnimatedValue();
            imageView.setImageResource(frames.get(index));
        });
        animator.setRepeatCount(ValueAnimator.INFINITE);
        animator.start();
    }

    public void stop() {
        if (animator != null) animator.cancel();
        animator = null;
    }
}
```

```java
// ✅ 方案二：Lottie 动画（推荐复杂动画场景）
```

### 5.4 其他常见坑

| 问题 | 原因 | 解决 |
|------|------|------|
| `ObjectAnimator` 不生效 | 目标属性没有 setter 或拼写错误 | 使用 `Property` 对象代替字符串 |
| 动画结束后 View 闪回初始状态 | `fillAfter` 仅对 View 动画有效 | 属性动画天然保持终态，无需设置 |
| 在子线程启动动画崩溃 | Choreographer 需要 Looper | 确保在主线程调用 `start()` |
| `setDuration` 对 AnimatorSet 子动画无效 | AnimatorSet.setDuration 会覆盖所有子动画 | 单独设置每个子动画的 duration |
| 硬件加速下动画闪烁 | 部分属性不支持硬件层 | 设置 `view.setLayerType(LAYER_TYPE_HARDWARE, null)` 或关闭硬件加速排查 |

### 5.5 性能优化建议

1. **优先使用 `ViewPropertyAnimator`**：`view.animate().translationX(100f).setDuration(300).start()` — 内部批量优化，性能最佳
2. **避免在 `onDraw()` 中创建动画对象**：会导致每帧创建新对象，GC 压力大
3. **使用硬件层加速复杂动画**：`view.setLayerType(LAYER_TYPE_HARDWARE, null)` 在动画期间开启，结束后关闭
4. **减少 `invalidate()` 范围**：属性动画自动调用 `invalidate()`，避免额外手动调用

```java
// ViewPropertyAnimator — 最简洁高效的动画 API
view.animate()
    .translationX(200f)
    .alpha(0.5f)
    .scaleX(1.5f)
    .setDuration(400)
    .setInterpolator(new OvershootInterpolator())
    .withEndAction(() -> { /* 动画结束回调 */ })
    .start();
```
