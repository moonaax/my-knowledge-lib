# 复合组件与自定义组合控件

> 将多个基础 View 组合成一个可复用的业务组件，是实际开发中最高频的自定义方式。
> 关联：[[3-自定义属性与TypedArray]] · [[1-View绘制流程-measure-layout-draw]] · [[5-事件分发与冲突处理]]

---

## 一、什么是复合组件

复合组件 = 把多个已有 View 组合成一个新组件，对外暴露统一的 API。

```
自定义方式（由简到难）：
1. 组合式：用 XML/Java 组合已有 View（本篇重点）
2. 扩展式：继承某个 View 添加功能（如 SmartTextView）
3. 完全自定义：继承 View/ViewGroup，重写 onDraw（前几篇讲的）
```

**实际开发中 80% 的自定义组件都是组合式**——比如：搜索栏（ImageView + EditText + Button）、标题栏（返回箭头 + 标题 + 更多按钮）、设置项（图标 + 标题 + 副标题 + 开关）。

## 二、实现方式

### 2.1 方式一：XML include + Java 绑定

最简单的方式，直接在布局中 include，Java 中 findViewById。

```java
// 布局文件：layout_search_bar.xml
// <LinearLayout>
//     <ImageView id="@+id/iv_icon" />
//     <EditText id="@+id/et_search" />
//     <Button id="@+id/btn_search" />
// </LinearLayout>

// 使用
View searchBar = LayoutInflater.from(context).inflate(R.layout.search_bar, this);
EditText etSearch = searchBar.findViewById(R.id.et_search);
```

**缺点**：每次使用都要 inflate + findViewById，不优雅。

### 2.2 方式二：自定义 ViewGroup 子类（推荐）

将布局加载和逻辑封装到一个类中，对外暴露简洁 API。

```java
public class SearchBar extends LinearLayout {

    private ImageView ivIcon;
    private EditText etSearch;
    private Button btnSearch;

    public SearchBar(Context context) {
        this(context, null);
    }

    public SearchBar(Context context, AttributeSet attrs) {
        this(context, attrs, 0);
    }

    public SearchBar(Context context, AttributeSet attrs, int defStyleAttr) {
        super(context, attrs, defStyleAttr);
        init(context, attrs);
    }

    private void init(Context context, AttributeSet attrs) {
        // 1. 加载布局
        inflate(context, R.layout.view_search_bar, this);
        // 2. 绑定子 View
        ivIcon = findViewById(R.id.iv_icon);
        etSearch = findViewById(R.id.et_search);
        btnSearch = findViewById(R.id.btn_search);
        // 3. 读取自定义属性
        if (attrs != null) {
            TypedArray ta = context.obtainStyledAttributes(attrs, R.styleable.SearchBar);
            String hint = ta.getString(R.styleable.SearchBar_searchHint);
            if (hint != null) etSearch.setHint(hint);
            ta.recycle();
        }
        // 4. 设置内部事件
        btnSearch.setOnClickListener(v -> {
            String query = etSearch.getText().toString().trim();
            if (onSearchListener != null) onSearchListener.onSearch(query);
        });
    }

    // ── 对外暴露的 API ──
    public String getQuery() {
        return etSearch.getText().toString().trim();
    }

    public void setQuery(String query) {
        etSearch.setText(query);
    }

    public void setHint(String hint) {
        etSearch.setHint(hint);
    }

    // ── 回调接口 ──
    private OnSearchListener onSearchListener;

    public void setOnSearchListener(OnSearchListener listener) {
        this.onSearchListener = listener;
    }

    public interface OnSearchListener {
        void onSearch(String query);
    }
}
```

**使用**：

```xml
<com.example.widget.SearchBar
    android:id="@+id/search_bar"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    app:searchHint="搜索知识库" />
```

```java
SearchBar searchBar = findViewById(R.id.search_bar);
searchBar.setOnSearchListener(query -> doSearch(query));
```

### 2.3 方式三：自定义 FrameLayout（更灵活）

当组件需要覆盖层（如角标、遮罩）时，FrameLayout 比 LinearLayout 更合适。

```java
public class BadgeImageView extends FrameLayout {

    private ImageView imageView;
    private TextView badgeView;

    public BadgeImageView(Context context, AttributeSet attrs) {
        super(context, attrs);
        init(context, attrs);
    }

    private void init(Context context, AttributeSet attrs) {
        // 图片
        imageView = new ImageView(context);
        addView(imageView, new LayoutParams(
            LayoutParams.MATCH_PARENT, LayoutParams.MATCH_PARENT));

        // 角标（右上角）
        badgeView = new TextView(context);
        badgeView.setBackgroundResource(R.drawable.bg_badge);
        badgeView.setTextColor(Color.WHITE);
        badgeView.setTextSize(10);
        badgeView.setGravity(Gravity.CENTER);
        badgeView.setVisibility(GONE);

        LayoutParams lp = new LayoutParams(
            dp2px(18), dp2px(18));
        lp.gravity = Gravity.TOP | Gravity.END;
        addView(badgeView, lp);

        // 读取属性
        if (attrs != null) {
            TypedArray ta = context.obtainStyledAttributes(attrs, R.styleable.BadgeImageView);
            int src = ta.getResourceId(R.styleable.BadgeImageView_badgeSrc, 0);
            if (src != 0) imageView.setImageResource(src);
            ta.recycle();
        }
    }

    public void setBadgeCount(int count) {
        if (count <= 0) {
            badgeView.setVisibility(GONE);
        } else {
            badgeView.setVisibility(VISIBLE);
            badgeView.setText(count > 99 ? "99+" : String.valueOf(count));
        }
    }

    private int dp2px(int dp) {
        return (int) (dp * getResources().getDisplayMetrics().density + 0.5f);
    }
}
```

## 三、自定义属性（attrs.xml）

复合组件必须定义自定义属性，否则使用者无法在 XML 中配置。

```xml
<!-- res/values/attrs.xml -->
<declare-styleable name="SearchBar">
    <attr name="searchHint" format="string" />
    <attr name="searchIcon" format="reference" />
    <attr name="showButton" format="boolean" />
</declare-styleable>

<declare-styleable name="BadgeImageView">
    <attr name="badgeSrc" format="reference" />
    <attr name="badgeCount" format="integer" />
</declare-styleable>
```

**format 类型速查**：

| format | 含义 | 示例 |
|--------|------|------|
| `string` | 字符串 | `app:title="设置"` |
| `integer` | 整数 | `app:maxLines="3"` |
| `boolean` | 布尔 | `app:showIcon="true"` |
| `color` | 颜色 | `app:textColor="#333"` |
| `dimension` | 尺寸 | `app:padding="16dp"` |
| `reference` | 资源引用 | `app:src="@drawable/icon"` |
| `enum` | 枚举 | `app:orientation="horizontal"` |
| `flag` | 位标志 | `app:gravity="center\|top"` |

## 四、实战：常见复合组件

### 4.1 设置项组件

```java
public class SettingItem extends LinearLayout {

    private ImageView ivIcon;
    private TextView tvTitle;
    private TextView tvDesc;
    private ImageView ivArrow;

    public SettingItem(Context context, AttributeSet attrs) {
        super(context, attrs);
        init(context, attrs);
    }

    private void init(Context context, AttributeSet attrs) {
        setOrientation(HORIZONTAL);
        setGravity(Gravity.CENTER_VERTICAL);
        int padding = dp2px(16);
        setPadding(padding, padding, padding, padding);

        inflate(context, R.layout.view_setting_item, this);
        ivIcon = findViewById(R.id.iv_icon);
        tvTitle = findViewById(R.id.tv_title);
        tvDesc = findViewById(R.id.tv_desc);
        ivArrow = findViewById(R.id.iv_arrow);

        if (attrs != null) {
            TypedArray ta = context.obtainStyledAttributes(attrs, R.styleable.SettingItem);
            ivIcon.setImageResource(ta.getResourceId(R.styleable.SettingItem_itemIcon, 0));
            tvTitle.setText(ta.getString(R.styleable.SettingItem_itemTitle));
            String desc = ta.getString(R.styleable.SettingItem_itemDesc);
            if (desc != null) {
                tvDesc.setText(desc);
                tvDesc.setVisibility(VISIBLE);
            }
            boolean showArrow = ta.getBoolean(R.styleable.SettingItem_showArrow, true);
            ivArrow.setVisibility(showArrow ? VISIBLE : GONE);
            ta.recycle();
        }
    }

    public void setTitle(String title) { tvTitle.setText(title); }
    public void setDesc(String desc) {
        tvDesc.setText(desc);
        tvDesc.setVisibility(VISIBLE);
    }
}
```

### 4.2 带输入框的对话框组件

```java
public class InputDialog extends Dialog {

    private EditText etInput;
    private Button btnConfirm;
    private Button btnCancel;

    public InputDialog(@NonNull Context context, String title) {
        super(context);
        setContentView(R.layout.dialog_input);

        tvTitle = findViewById(R.id.tv_title);
        etInput = findViewById(R.id.et_input);
        btnConfirm = findViewById(R.id.btn_confirm);
        btnCancel = findViewById(R.id.btn_cancel);

        tvTitle.setText(title);

        btnCancel.setOnClickListener(v -> dismiss());
    }

    public void setOnConfirmListener(OnConfirmListener listener) {
        btnConfirm.setOnClickListener(v -> {
            String input = etInput.getText().toString().trim();
            if (!input.isEmpty()) {
                listener.onConfirm(input);
                dismiss();
            }
        });
    }

    public interface OnConfirmListener {
        void onConfirm(String input);
    }
}
```

## 五、复合组件的最佳实践

### 5.1 三构造函数规范

```java
// 必须实现三个构造函数，保证 XML 和 Java 创建都能用
public MyView(Context context) {
    this(context, null);  // Java 代码创建
}

public MyView(Context context, AttributeSet attrs) {
    this(context, attrs, 0);  // XML 中使用
}

public MyView(Context context, AttributeSet attrs, int defStyleAttr) {
    super(context, attrs, defStyleAttr);
    init(context, attrs);  // 统一初始化
}
```

### 5.2 LayoutParams 规范

```java
// 继承 LinearLayout → 子 View 用 LinearLayout.LayoutParams
// 继承 FrameLayout → 子 View 用 FrameLayout.LayoutParams
// 继承 RelativeLayout → 子 View 用 RelativeLayout.LayoutParams
// 不要混用！
```

### 5.3 对外 API 设计原则

| 原则 | 说明 |
|------|------|
| 最小暴露 | 只暴露必要的 get/set，内部实现细节不暴露 |
| 回调统一 | 用 Listener 模式，不用 public 方法让外部轮询 |
| 空安全 | set 方法要处理 null，get 方法可能返回 null |
| 线程安全 | UI 操作必须在主线程，set 方法内部 post 到主线程 |

## 六、面试要点

**Q: 复合组件和自定义 View 的区别？**

复合组件是把多个已有 View 组合成一个新组件，不需要重写 onDraw；自定义 View 需要自己处理测量、布局、绘制。复合组件开发效率高，适合业务组件；自定义 View 灵活度高，适合特殊 UI 效果。

**Q: 为什么要重写三个构造函数？**

单参数构造用于 Java 代码创建，双参数构造用于 XML inflate，三参数构造支持 defStyleAttr 主题覆盖。三个构造函数链式调用，确保无论哪种方式创建都能正确初始化。

**Q: 如何让复合组件支持自定义属性？**

在 attrs.xml 中声明 declare-styleable，在三参数构造中通过 TypedArray 读取属性值，记得调用 recycle() 释放。对外暴露对应的 set 方法，支持运行时修改。

**Q: 复合组件的 LayoutParams 有什么坑？**

子 View 的 LayoutParams 必须和父容器类型匹配。继承 LinearLayout 就用 LinearLayout.LayoutParams，不能用 FrameLayout.LayoutParams，否则会抛 ClassCastException。inflate 子布局时也要注意 parent 参数。
