# 什么是 AI Agent

## 1. 定义

AI Agent（智能体）是一个能够**感知环境、自主决策、执行行动**的系统。与传统的 AI 模型不同，Agent 不仅仅是"给输入返回输出"，而是能够：

- 理解复杂目标并拆解为子任务
- 自主选择工具和策略
- 根据反馈动态调整行为
- 在多轮交互中保持上下文

> **一句话理解**：LLM 是大脑，Agent 是拥有大脑、手脚和记忆的完整个体。

## 2. Agent 的核心组成

````
┌─────────────────────────────────────┐
│              AI Agent               │
├─────────────────────────────────────┤
│  🧠 大脑 (LLM)                      │
│     - 理解、推理、规划、生成          │
├─────────────────────────────────────┤
│  🔧 工具 (Tools)                     │
│     - API 调用、代码执行、搜索引擎    │
│     - 数据库查询、文件操作            │
├─────────────────────────────────────┤
│  💾 记忆 (Memory)                    │
│     - 短期记忆：当前对话上下文         │
│     - 长期记忆：向量数据库、知识库     │
├─────────────────────────────────────┤
│  📋 规划 (Planning)                  │
│     - 任务分解、策略选择、自我反思     │
└─────────────────────────────────────┘
````
### 2.1 大脑（LLM）

Agent 的核心推理引擎，负责：
- **理解**：解析用户意图和上下文
- **推理**：逻辑推导和因果分析
- **规划**：制定执行计划
- **生成**：产出文本、代码、决策

### 2.2 工具（Tools）

Agent 与外部世界交互的接口：

````python
# 工具定义示例
tools = [
    {
        "name": "search_web",
        "description": "搜索互联网获取最新信息",
        "parameters": {
            "query": {"type": "string", "description": "搜索关键词"}
        }
    },
    {
        "name": "execute_code",
        "description": "执行 Python 代码并返回结果",
        "parameters": {
            "code": {"type": "string", "description": "要执行的代码"}
        }
    }
]
````
### 2.3 记忆（Memory）

| 类型   | 存储方式  | 生命周期 | 典型用途      |
| ---- | ----- | ---- | --------- |
| 短期记忆 | 对话上下文 | 单次会话 | 多轮对话连贯性   |
| 长期记忆 | 向量数据库 | 持久化  | 用户偏好、历史知识 |
| 工作记忆 | 中间状态  | 单次任务 | 任务执行中间结果  |

### 2.4 规划（Planning）

Agent 的规划能力决定了它能处理多复杂的任务：

````
用户目标: "帮我分析这个项目的代码质量并生成报告"

Agent 规划:
  1. 读取项目结构 → 使用文件系统工具
  2. 分析代码复杂度 → 执行静态分析工具
  3. 检查测试覆盖率 → 运行测试框架
  4. 汇总结果 → 生成 Markdown 报告
  5. 自我检查 → 验证报告完整性
````
## 3. Agent vs 传统 AI 应用

| 维度    | 传统 AI（ChatBot） | AI Agent |
| ----- | -------------- | -------- |
| 交互模式  | 单轮问答           | 多轮自主执行   |
| 工具使用  | 无              | 动态调用多种工具 |
| 任务复杂度 | 简单问答           | 复杂多步骤任务  |
| 自主性   | 被动响应           | 主动规划和执行  |
| 错误处理  | 无              | 自我反思和纠错  |
| 记忆    | 有限上下文窗口        | 短期+长期记忆  |

## 4. Agent 的发展阶段

````
Level 1: 简单对话 → ChatGPT 式问答
Level 2: 工具调用 → Function Calling，调用 API
Level 3: 自主规划 → ReAct、Plan-and-Execute
Level 4: 多 Agent 协作 → 多个 Agent 分工合作
Level 5: 自我进化 → Agent 能自我改进和学习
````
## 5. 典型 Agent 应用场景

### 开发助手
- 代码生成、Review、Debug
- 自动化测试、CI/CD 集成
- 文档生成和维护

### 数据分析
- 自然语言查询数据库
- 自动生成可视化报告
- 异常检测和预警

### 自动化办公
- 邮件处理和回复
- 会议纪要生成
- 项目管理和任务分配

### 客户服务
- 智能客服（带知识库检索）
- 工单自动分类和路由
- 问题自动诊断

## 7. 关键术语速查

| 术语                    | 含义                                    |
| --------------------- | ------------------------------------- |
| LLM                   | Large Language Model，大语言模型            |
| Agent                 | 能自主决策和执行的 AI 系统                       |
| Tool/Function Calling | Agent 调用外部工具的能力                       |

## 面试题精选

### Q1: AI Agent 和传统 ChatBot 的核心区别是什么？
**答：** Agent 具备自主规划、工具调用和记忆能力，能处理多步骤复杂任务并根据反馈动态调整策略；ChatBot 只是单轮或简单多轮的被动问答，没有工具使用和自主决策能力。

### Q2: Agent 的四个核心组成部分分别是什么？各自的作用？
**答：** 大脑（LLM）负责理解、推理和生成；工具（Tools）提供与外部世界交互的能力如 API 调用、代码执行；记忆（Memory）维持对话连贯性和长期知识积累；规划（Planning）负责任务分解和策略选择。

### Q3: Agent 的短期记忆和长期记忆有什么区别？分别怎么实现？
**答：** 短期记忆是当前会话的对话上下文，通常直接存在消息列表中，生命周期为单次会话。长期记忆是跨会话持久化的知识，通常用向量数据库存储，通过语义检索召回相关记忆。

### Q4: 为什么说 Agent 需要工具调用能力？LLM 本身有什么不足？
**答：** LLM 不擅长精确数学计算、无法获取实时信息、存在幻觉问题、受上下文窗口限制。通过工具调用（搜索引擎、计算器、代码执行器等），Agent 可以弥补这些不足，让 LLM 专注于推理和决策。

### Q5: Agent 的发展可以分为哪几个阶段？当前主流处于哪个阶段？
**答：** 从简单对话（L1）→ 工具调用（L2）→ 自主规划（L3）→ 多 Agent 协作（L4）→ 自我进化（L5）。当前主流应用处于 L2-L3 阶段，L4 多 Agent 协作正在快速发展中。

### Q6: 设计一个 Agent 系统时，最需要关注的技术挑战有哪些？
**答：** 主要挑战包括：规划能力的可靠性（避免死循环或偏离目标）、工具调用的准确性（参数生成正确性）、记忆管理的效率（上下文窗口限制）、幻觉控制、成本控制（Token 消耗）以及安全性（Prompt 注入防护）。
