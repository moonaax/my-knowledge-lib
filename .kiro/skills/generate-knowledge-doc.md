# Skill: 生成知识文档

## 描述
根据指定主题，在对应模块目录下生成一篇结构化的知识文档（Markdown），遵循知识库现有的写作风格和格式规范。

## 触发条件
用户要求生成/编写/新增某个主题的知识文档时触发。

## 输入
- `topic`: 文档主题（如"协程原理"、"Embedding模型选型"）
- `module`: 所属模块路径（如"Android知识/性能优化"、"Agent AI学习/RAG检索增强生成"）

## 执行步骤

1. **确认模块目录存在**：检查 `{module}` 目录是否存在，不存在则创建并提示用户。

2. **分析现有文档风格**：读取同模块下 1-2 篇已有文档，提取写作风格：
   - 一级标题 = 文档主题
   - 开头有 `>` 引用块做简介
   - 有 `相关主题：[[xxx]] | [[yyy]]` 的双链引用
   - 用 `---` 分隔引言和正文
   - 正文用 `## 一、` `## 二、` 编号章节，子节用 `### 1.1` 格式
   - 善用表格对比、代码块示例、`> 🔑` 关键提示
   - 内容深度：原理 + 源码分析 + 实战 + 面试要点

3. **生成文档**：按以下结构生成内容：
   ```
   # {topic}
   > 一句话简介
   相关主题：[[相关1]] | [[相关2]]
   ---
   ## 一、核心概念
   ## 二、原理分析（含源码/流程图）
   ## 三、实战应用（含代码示例）
   ## 四、常见问题与面试要点
   ## 五、面试题精选（5-10 题，含简要解答）
   ## 六、总结与参考
   ```

4. **写入文件**：保存到 `{module}/{topic}.md`。

5. **更新模块 README**：在 `{module}/README.md` 中追加 `- [[{topic}]]` 链接。

## 约束
- 文档内容不少于 3000 字，确保有深度
- Android 模块代码示例优先使用 Java，能用 Java 就不用 Kotlin；Agent AI 模块使用 Python
- 文档末尾必须包含 `## 面试要点` 章节，列出 5-10 个高频面试题及简要解答
- 使用 Obsidian 双链语法 `[[]]` 关联相关文档
- 不要生成与已有文档重复的内容，先检查是否已存在同名文件
