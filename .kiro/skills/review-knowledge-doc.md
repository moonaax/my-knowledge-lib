# Skill: 审查知识文档

## 描述
审查并优化已有的知识文档，检查结构完整性、内容深度、格式规范，输出改进建议或直接修复。

## 触发条件
用户要求审查/检查/优化某篇或某模块的知识文档时触发。

## 输入
- `target`: 目标文件路径或模块目录（如"Android知识/性能优化/启动优化.md"或"Agent AI学习/基础概念"）

## 执行步骤

1. **读取目标文档**：如果是目录则遍历所有 `.md` 文件（排除 README.md）。

2. **逐篇检查**，对照以下清单：

   | 检查项 | 标准 |
   |-------|------|
   | 标题结构 | 有 `# 主题`，章节用 `## 一、` 编号 |
   | 引言 | 开头有 `>` 引用块简介 |
   | 相关链接 | 有 `相关主题：[[xx]]` 双链 |
   | 分隔线 | 引言后有 `---` |
   | 内容深度 | 包含原理分析、代码示例、实战场景 |
   | 代码块 | 有语言标注（kotlin/java/python 等） |
   | 关键提示 | 适当使用 `> 🔑` 标记重点 |
   | 字数 | 不少于 3000 字 |
   | 双链引用 | `[[]]` 链接指向的文档确实存在 |

3. **输出审查报告**：按文件列出问题和建议，格式：
   ```
   ## 📋 审查报告：{文件名}
   - ✅ 标题结构：正常
   - ⚠️ 缺少相关主题链接
   - ❌ 代码块未标注语言（第 45 行）
   ```

4. **自动修复（可选）**：用户确认后，自动修复格式类问题（补分隔线、补代码语言标注等）。

## 约束
- 不擅自修改文档内容观点，仅修复格式和结构问题
- 修复前必须向用户展示审查报告并获得确认
- 检查双链时只验证文件是否存在，不验证内容
