# 数组与哈希

## 核心技巧

- HashMap 空间换时间，O(n) 解决两数之和类问题
- 原地哈希：用数组下标当 key，O(1) 空间
- 前缀和：快速求区间和
- 排序预处理：简化去重、分组逻辑

---

## 1. 两数之和（#1）

> 给定数组和目标值，找两个数的下标使其和等于 target

**思路**：一遍哈希，边遍历边查 `target - nums[i]` 是否已存在

```java
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement)) {
            return new int[]{map.get(complement), i};
        }
        map.put(nums[i], i);
    }
    return new int[0];
}
```

- 时间 O(n)，空间 O(n)
- 踩坑：先查再存，避免用自身配对

**📝 解析**

- **思路推导**：暴力双重循环 O(n²)，用哈希表把"查找 complement 是否存在"降到 O(1)。一边遍历一边存，保证不会用同一个元素配对自己。
- **关键点**：`map.put` 放在 `if` 判断之后，这样 `[3,3] target=6` 时第二个 3 能找到第一个 3。
- **易错点**：如果先 put 再查，`[3,2,4] target=6` 中 nums[0]=3 会和自己配对返回 `[0,0]`。
- **举一反三**：
  - 三数之和（#15）→ 固定一个数后转化为两数之和
  - 四数之和（#18）→ 再多一层循环
  - 两数之和变体：返回布尔值 / 返回所有配对 / 有序数组用双指针

## 2. 字母异位词分组（#49）

> 把字母异位词分到一组

**思路**：排序后的字符串作为 key

```java
public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    for (String s : strs) {
        char[] arr = s.toCharArray();
        Arrays.sort(arr);
        String key = new String(arr);
        map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
    }
    return new ArrayList<>(map.values());
}
```

- 时间 O(n·k·log k)，k 为字符串平均长度
- 技巧：也可用 26 位计数数组做 key，避免排序

**📝 解析**

- **思路推导**：异位词排序后相同，所以排序结果可以作为分组的 key。问题转化为"按 key 分组"，天然适合 HashMap。
- **关键点**：`computeIfAbsent` 简化了"key 不存在则创建新 list"的逻辑。
- **易错点**：用 `char[]` 做 key 不行（数组的 equals 比较的是引用），必须转成 `String`。
- **举一反三**：
  - 计数数组做 key：`"1#0#2#..."` 格式，时间降到 O(n·k)，适合字符串很长的场景
  - 有效的字母异位词（#242）→ 本题的子问题

## 3. 最长连续序列（#128）

> 找无序数组中最长连续序列的长度

**思路**：HashSet 去重，只从序列起点开始计数

```java
public int longestConsecutive(int[] nums) {
    Set<Integer> set = new HashSet<>();
    for (int n : nums) set.add(n);
    int max = 0;
    for (int n : set) {
        if (!set.contains(n - 1)) { // 只从起点开始
            int len = 1;
            while (set.contains(n + len)) len++;
            max = Math.max(max, len);
        }
    }
    return max;
}
```

- 时间 O(n)，空间 O(n)
- 关键：`!set.contains(n-1)` 保证每个序列只遍历一次

**📝 解析**

- **思路推导**：排序后遍历是 O(n·log n)，要做到 O(n) 就需要哈希。关键洞察：只从序列的起点（即 n-1 不在集合中）开始向后数，避免重复计算。
- **关键点**：虽然有嵌套循环，但每个元素最多被 while 访问一次，总时间仍是 O(n)。
- **易错点**：不加 `!set.contains(n-1)` 判断会退化为 O(n²)，比如 `[1,2,3,...,n]`。
- **举一反三**：
  - 类似"连续"问题都可以考虑 HashSet + 起点判断
  - 并查集也能解，但代码更复杂

## 4. 有效的字母异位词（#242）

> 判断两个字符串是否是字母异位词

**思路**：26 位计数数组

```java
public boolean isAnagram(String s, String t) {
    if (s.length() != t.length()) return false;
    int[] count = new int[26];
    for (int i = 0; i < s.length(); i++) {
        count[s.charAt(i) - 'a']++;
        count[t.charAt(i) - 'a']--;
    }
    for (int c : count) {
        if (c != 0) return false;
    }
    return true;
}
```

- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：排序后比较是 O(n·log n)，用计数数组只需 O(n)。一个字符串加，另一个减，最终全 0 就是异位词。
- **关键点**：只有小写字母时用 `int[26]` 即可；如果包含 Unicode 字符则需要 HashMap。
- **易错点**：先判断长度不等直接返回 false，避免无意义计算。
- **举一反三**：字母异位词分组（#49）→ 本题的批量版本

## 5. 存在重复元素（#217）

> 判断数组中是否有重复元素

```java
public boolean containsDuplicate(int[] nums) {
    Set<Integer> set = new HashSet<>();
    for (int n : nums) {
        if (!set.add(n)) return true;
    }
    return false;
}
```

- 时间 O(n)，空间 O(n)
- 技巧：`set.add()` 返回 false 说明已存在

**📝 解析**

- **思路推导**：排序后相邻比较是 O(n·log n)，HashSet 判重是 O(n)。利用 `add()` 的返回值一行搞定。
- **关键点**：`HashSet.add()` 返回 boolean，元素已存在时返回 false，比先 contains 再 add 更简洁。
- **举一反三**：
  - 存在重复元素 II（#219）→ 加上距离约束，用 HashMap 存下标
  - 存在重复元素 III（#220）→ 桶排序思想

## 6. 除自身以外数组的乘积（#238）

> 不用除法，返回每个位置其余元素的乘积

**思路**：前缀积 × 后缀积

```java
public int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] res = new int[n];
    res[0] = 1;
    for (int i = 1; i < n; i++) {
        res[i] = res[i - 1] * nums[i - 1]; // 前缀积
    }
    int suffix = 1;
    for (int i = n - 2; i >= 0; i--) {
        suffix *= nums[i + 1]; // 后缀积
        res[i] *= suffix;
    }
    return res;
}
```

- 时间 O(n)，空间 O(1)（结果数组不算额外空间）

**📝 解析**

- **思路推导**：不能用除法，那就拆成"左边所有数的乘积 × 右边所有数的乘积"。第一遍从左到右算前缀积存入 res，第二遍从右到左乘上后缀积。
- **关键点**：用 res 数组先存前缀积，再原地乘后缀积，避免额外数组。
- **易错点**：第二遍 suffix 初始值为 1，从倒数第二个开始（最后一个元素的后缀积就是 1）。
- **举一反三**：前缀和/前缀积是一类通用技巧，和为 K 的子数组（#560）也用了类似思想。

## 7. 前 K 个高频元素（#347）

> 返回出现频率前 k 高的元素

**思路**：桶排序，下标是频率

```java
public int[] topKFrequent(int[] nums, int k) {
    Map<Integer, Integer> freq = new HashMap<>();
    for (int n : nums) freq.merge(n, 1, Integer::sum);

    List<Integer>[] buckets = new List[nums.length + 1];
    for (var entry : freq.entrySet()) {
        int f = entry.getValue();
        if (buckets[f] == null) buckets[f] = new ArrayList<>();
        buckets[f].add(entry.getKey());
    }

    int[] res = new int[k];
    int idx = 0;
    for (int i = buckets.length - 1; i >= 0 && idx < k; i--) {
        if (buckets[i] != null) {
            for (int n : buckets[i]) {
                res[idx++] = n;
                if (idx == k) break;
            }
        }
    }
    return res;
}
```

- 时间 O(n)，空间 O(n)
- 也可用小顶堆 O(n·log k)，但桶排序更优

**📝 解析**

- **思路推导**：统计频率后需要找 Top K。三种方案：① 排序 O(n·log n) ② 小顶堆 O(n·log k) ③ 桶排序 O(n)。桶排序以频率为下标，频率最大为 n，从后往前取 k 个。
- **关键点**：桶的大小是 `nums.length + 1`（频率范围 0~n），下标就是频率值。
- **易错点**：同一频率可能有多个元素，所以桶里存的是 List。取 k 个时注意一个桶里可能有多个。
- **举一反三**：
  - 前 K 个高频单词（#692）→ 同频时按字典序，需要 TreeMap 或自定义比较器
  - 快速选择算法（QuickSelect）也能 O(n) 平均解决 Top K

## 8. 和为 K 的子数组（#560）

> 找连续子数组和等于 k 的个数

**思路**：前缀和 + HashMap

```java
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> map = new HashMap<>();
    map.put(0, 1); // 前缀和为 0 出现 1 次
    int sum = 0, count = 0;
    for (int n : nums) {
        sum += n;
        count += map.getOrDefault(sum - k, 0);
        map.merge(sum, 1, Integer::sum);
    }
    return count;
}
```

- 时间 O(n)，空间 O(n)
- 核心：`sum[i..j] = prefix[j] - prefix[i-1] = k`

**📝 解析**

- **思路推导**：暴力枚举所有子数组 O(n²)。用前缀和优化：`prefix[j] - prefix[i] = k` 等价于 `prefix[i] = prefix[j] - k`。用 HashMap 记录每个前缀和出现的次数，遍历到 j 时查 `sum - k` 出现了几次。
- **关键点**：`map.put(0, 1)` 处理从下标 0 开始的子数组（前缀和本身等于 k 的情况）。
- **易错点**：不能用滑动窗口！因为数组有负数，窗口扩大/缩小时和不单调。
- **举一反三**：
  - 路径总和 III（#437）→ 树上的前缀和，思路完全一样
  - 连续的子数组和（#523）→ 前缀和 mod k

## 9. 合并区间（#56）

> 合并所有重叠区间

**思路**：按起点排序，逐个合并

```java
public int[][] merge(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
    List<int[]> res = new ArrayList<>();
    res.add(intervals[0]);
    for (int i = 1; i < intervals.length; i++) {
        int[] last = res.get(res.size() - 1);
        if (intervals[i][0] <= last[1]) {
            last[1] = Math.max(last[1], intervals[i][1]);
        } else {
            res.add(intervals[i]);
        }
    }
    return res.toArray(new int[0][]);
}
```

- 时间 O(n·log n)，空间 O(n)

**📝 解析**

- **思路推导**：排序后重叠区间一定相邻。遍历时只需比较当前区间的起点和上一个区间的终点：起点 ≤ 终点就合并（取更大的终点），否则新开一个区间。
- **关键点**：合并时更新 `last[1] = max(last[1], intervals[i][1])`，不能直接赋值，因为可能出现 `[1,4]` 和 `[2,3]` 的包含关系。
- **易错点**：排序比较器 `(a,b) -> a[0] - b[0]` 在极端值时可能溢出，更安全的写法是 `Integer.compare(a[0], b[0])`。
- **举一反三**：
  - 插入区间（#57）→ 先插入再合并
  - 无重叠区间（#435）→ 按右端点排序的贪心

## 10. 缺失的第一个正数（#41）

> 找未排序数组中缺失的最小正整数，要求 O(n) 时间 O(1) 空间

**思路**：原地哈希，把 nums[i] 放到下标 nums[i]-1 的位置

```java
public int firstMissingPositive(int[] nums) {
    int n = nums.length;
    for (int i = 0; i < n; i++) {
        while (nums[i] > 0 && nums[i] <= n && nums[nums[i] - 1] != nums[i]) {
            int tmp = nums[nums[i] - 1];
            nums[nums[i] - 1] = nums[i];
            nums[i] = tmp;
        }
    }
    for (int i = 0; i < n; i++) {
        if (nums[i] != i + 1) return i + 1;
    }
    return n + 1;
}
```

- 时间 O(n)，空间 O(1)
- 踩坑：while 循环条件要判断 `nums[nums[i]-1] != nums[i]` 防死循环

**📝 解析**

- **思路推导**：O(n) 时间 + O(1) 空间意味着不能用额外哈希表。核心思想：把数组本身当哈希表，让 `nums[i] = i + 1`。遍历时把每个在 [1, n] 范围内的数交换到正确位置。
- **关键点**：用 while 而不是 if，因为交换过来的新数可能也需要归位。条件 `nums[nums[i]-1] != nums[i]` 防止相同值死循环。
- **易错点**：交换时注意先算目标位置 `nums[i]-1`，不要写反。负数和超出范围的数直接跳过。
- **举一反三**：
  - 找到所有数组中消失的数字（#448）→ 同样的原地哈希思想
  - 数组中重复的数据（#442）→ 原地标记法

## 11. 多数元素（#169）

> 找出现次数超过 n/2 的元素

**思路**：摩尔投票法

```java
public int majorityElement(int[] nums) {
    int candidate = 0, count = 0;
    for (int n : nums) {
        if (count == 0) candidate = n;
        count += (n == candidate) ? 1 : -1;
    }
    return candidate;
}
```

- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：HashMap 计数是 O(n) 时间 O(n) 空间。摩尔投票法做到 O(1) 空间：把不同的元素两两抵消，最后剩下的就是多数元素（因为它超过 n/2）。
- **关键点**：`count == 0` 时换候选人，相当于前面的元素已经两两抵消完了。
- **易错点**：本题保证多数元素一定存在。如果不保证，需要第二遍验证候选人的出现次数。
- **举一反三**：
  - 多数元素 II（#229）→ 找出现超过 n/3 的元素，最多 2 个，用两组候选人

## 12. 移动零（#283）

> 把所有 0 移到末尾，保持非零元素相对顺序

```java
public void moveZeroes(int[] nums) {
    int slow = 0;
    for (int fast = 0; fast < nums.length; fast++) {
        if (nums[fast] != 0) {
            nums[slow++] = nums[fast];
        }
    }
    while (slow < nums.length) nums[slow++] = 0;
}
```

- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：快慢指针思想。slow 指向下一个非零元素应该放的位置，fast 遍历数组。非零元素依次覆盖到前面，最后把 slow 之后的位置填 0。
- **关键点**：这本质上是"删除特定元素"的变体（删除 0），和删除有序数组重复项（#26）思路一致。
- **易错点**：也可以用 swap 写法（遇到非零就和 slow 交换），但覆盖+填零更简洁。
- **举一反三**：移除元素（#27）→ 完全相同的模板

## 13. 旋转数组（#189）

> 将数组向右旋转 k 步

**思路**：三次翻转

```java
public void rotate(int[] nums, int k) {
    k %= nums.length;
    reverse(nums, 0, nums.length - 1);
    reverse(nums, 0, k - 1);
    reverse(nums, k, nums.length - 1);
}

private void reverse(int[] nums, int l, int r) {
    while (l < r) {
        int tmp = nums[l];
        nums[l++] = nums[r];
        nums[r--] = tmp;
    }
}
```

- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：额外数组很简单但空间 O(n)。三次翻转法：先整体翻转，再分别翻转前 k 个和后 n-k 个。例如 `[1,2,3,4,5] k=2` → 整体翻转 `[5,4,3,2,1]` → 翻转前2 `[4,5,3,2,1]` → 翻转后3 `[4,5,1,2,3]`。
- **关键点**：`k %= nums.length` 处理 k 大于数组长度的情况。
- **易错点**：翻转的边界要准确：`[0, k-1]` 和 `[k, n-1]`。
- **举一反三**：翻转字符串里的单词（#151）→ 也是先整体翻转再局部翻转

## 14. 螺旋矩阵（#54）

> 按螺旋顺序返回矩阵所有元素

```java
public List<Integer> spiralOrder(int[][] matrix) {
    List<Integer> res = new ArrayList<>();
    int top = 0, bottom = matrix.length - 1;
    int left = 0, right = matrix[0].length - 1;
    while (top <= bottom && left <= right) {
        for (int i = left; i <= right; i++) res.add(matrix[top][i]);
        top++;
        for (int i = top; i <= bottom; i++) res.add(matrix[i][right]);
        right--;
        if (top <= bottom) {
            for (int i = right; i >= left; i--) res.add(matrix[bottom][i]);
            bottom--;
        }
        if (left <= right) {
            for (int i = bottom; i >= top; i--) res.add(matrix[i][left]);
            left++;
        }
    }
    return res;
}
```

- 时间 O(m×n)，空间 O(1)
- 踩坑：内层两个 for 要加边界判断，防止单行/单列重复遍历

**📝 解析**

- **思路推导**：维护四个边界 top/bottom/left/right，按"右→下→左→上"的顺序遍历，每走完一条边就收缩对应边界。
- **关键点**：向左和向上遍历前要检查 `top <= bottom` 和 `left <= right`，因为前两步可能已经把行/列遍历完了。
- **易错点**：单行矩阵 `[[1,2,3]]` 或单列矩阵如果不加判断会重复遍历。
- **举一反三**：螺旋矩阵 II（#59）→ 同样的边界收缩，改为填数

## 15. 矩阵置零（#73）

> 如果元素为 0，将其所在行列全部置零

**思路**：用第一行第一列做标记

```java
public void setZeroes(int[][] matrix) {
    int m = matrix.length, n = matrix[0].length;
    boolean firstRow = false, firstCol = false;
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (matrix[i][j] == 0) {
                if (i == 0) firstRow = true;
                if (j == 0) firstCol = true;
                matrix[i][0] = 0;
                matrix[0][j] = 0;
            }
        }
    }
    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            if (matrix[i][0] == 0 || matrix[0][j] == 0)
                matrix[i][j] = 0;
    if (firstRow) for (int j = 0; j < n; j++) matrix[0][j] = 0;
    if (firstCol) for (int i = 0; i < m; i++) matrix[i][0] = 0;
}
```

- 时间 O(m×n)，空间 O(1)

**📝 解析**

- **思路推导**：O(m+n) 空间很简单（用两个数组记录哪些行列要置零）。O(1) 空间的关键：用矩阵的第一行和第一列本身作为标记数组。但第一行/第一列自身是否要置零需要额外两个布尔变量记录。
- **关键点**：置零的顺序很重要——先处理非首行首列（从 `[1][1]` 开始），最后再处理首行首列，否则标记会被提前覆盖。
- **易错点**：`firstRow` 和 `firstCol` 的判断要在标记阶段同步完成，不能放到后面。
- **举一反三**：这种"用数据本身做标记"的思想在缺失的第一个正数（#41）中也有体现。
