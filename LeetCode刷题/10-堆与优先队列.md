# 堆与优先队列

## 核心技巧

- 堆 = 完全二叉树，Java 的 `PriorityQueue` 默认小顶堆
- 大顶堆：`Collections.reverseOrder()` 或 `(a, b) -> b - a`
- 适用场景：TopK 问题、动态中位数、合并多个有序序列
- 堆的核心操作：插入 O(log n)、取堆顶 O(1)、删除堆顶 O(log n)
- 手写堆：数组表示，`parent = (i-1)/2`，`left = 2i+1`，`right = 2i+2`

## 1. 数组中的第 K 个最大元素（#215）

> 找数组中第 K 大的数（不是第 K 大的不同元素）

**思路**：维护大小为 K 的小顶堆，遍历完后堆顶就是第 K 大

```java
public int findKthLargest(int[] nums, int k) {
    PriorityQueue<Integer> minHeap = new PriorityQueue<>(k);
    for (int n : nums) {
        minHeap.offer(n);
        if (minHeap.size() > k) minHeap.poll();
    }
    return minHeap.peek();
}
```

- 时间 O(n·log k)，空间 O(k)
- 比排序 O(n·log n) 快

**📝 解析**

- **思路推导**：排序后取第 K 个 O(n·log n)。优化：维护大小为 K 的小顶堆，堆顶是堆中最小的 = 第 K 大的。新元素比堆顶大就替换，否则跳过。遍历完堆顶就是答案。
- **关键点**：为什么是小顶堆不是大顶堆？因为我们要保留"最大的 K 个"，小顶堆的堆顶是这 K 个中最小的——新元素只要比它大就该入堆。
- **易错点**：`minHeap.size() > k` 时 poll，保持堆大小恰好为 K。如果用大顶堆就只能一个个 poll K 次，效率一样但写法不同。
- **举一反三**：
  - 快速选择算法（QuickSelect）：平均 O(n)，最坏 O(n²)，面试也能说
  - 前 K 个高频元素（#347）→ 同样的 TopK 模式
  - 数据流的中位数（#295）→ 两个堆配合

## 2. 前 K 个高频元素（#347）

> 返回出现频率最高的 K 个元素

**思路**：HashMap 统计频率 → 小顶堆取 TopK

```java
public int[] topKFrequent(int[] nums, int k) {
    // 1. 统计频率
    Map<Integer, Integer> freq = new HashMap<>();
    for (int n : nums) freq.merge(n, 1, Integer::sum);

    // 2. 小顶堆，按频率排序
    PriorityQueue<int[]> heap = new PriorityQueue<>((a, b) -> a[1] - b[1]);
    for (Map.Entry<Integer, Integer> e : freq.entrySet()) {
        heap.offer(new int[]{e.getKey(), e.getValue()});
        if (heap.size() > k) heap.poll();
    }

    // 3. 取结果
    int[] res = new int[k];
    for (int i = 0; i < k; i++) res[i] = heap.poll()[0];
    return res;
}
```

- 时间 O(n·log k)，空间 O(n)
- 堆里存 [元素, 频率] 对

**📝 解析**

- **思路推导**：先 HashMap 统计每个元素出现次数。然后问题变成"从频率表中找频率最大的 K 个" = TopK 问题，用小顶堆。
- **关键点**：堆的比较器用 `a[1] - b[1]`（按频率比较），不是按元素值。堆里存 int[]{元素, 频率}。
- **易错点**：`freq.merge(n, 1, Integer::sum)` 比 `getOrDefault + put` 更简洁。堆 poll 出来的顺序是频率从小到大，如果要求频率从大到小需要反转。
- **举一反三**：
  - 桶排序解法：频率作为桶下标，O(n) 时间，面试能说更优解
  - 重排字符串（#767）→ 用频率 + 贪心/堆

## 3. 数据流的中位数（#295）

> 动态添加数字，随时查询中位数

**思路**：大顶堆存较小一半 + 小顶堆存较大一半，保持平衡

```java
class MedianFinder {
    private PriorityQueue<Integer> small; // 大顶堆，存较小的一半
    private PriorityQueue<Integer> large; // 小顶堆，存较大的一半

    public MedianFinder() {
        small = new PriorityQueue<>(Collections.reverseOrder());
        large = new PriorityQueue<>();
    }

    public void addNum(int num) {
        small.offer(num);
        large.offer(small.poll());  // 把 small 的最大值给 large
        if (large.size() > small.size()) {
            small.offer(large.poll());  // 保持 small >= large
        }
    }

    public double findMedian() {
        if (small.size() > large.size()) return small.peek();
        return (small.peek() + large.peek()) / 2.0;
    }
}
```

- addNum O(log n)，findMedian O(1)
- 核心：small 的所有元素 ≤ large 的所有元素

**📝 解析**

- **思路推导**：排序后取中位数 O(n·log n)。优化：用两个堆把数据分成两半——small（大顶堆）存较小的一半，large（小顶堆）存较大的的一半。中位数就是 small 堆顶或两个堆顶的平均。
- **关键点**：addNum 的三步操作——①先入 small ②把 small 最大值给 large ③如果 large 太大就还一个给 small。这样保证了 small 的所有元素 ≤ large 的所有元素，且 size 差 ≤ 1。
- **易错点**：偶数个元素时中位数是两个堆顶的平均值，不是 `large.peek()`。`/ 2.0` 不能写成 `/ 2`（整数除法）。
- **举一反三**：滑动窗口中位数（#480）→ 困难版，需要延迟删除

## 4. 合并 K 个升序链表（#23）

> K 个已排序链表，合并成一个排序链表

**思路**：小顶堆存每个链表的当前头节点，每次取最小的

```java
public ListNode mergeKLists(ListNode[] lists) {
    PriorityQueue<ListNode> heap = new PriorityQueue<>((a, b) -> a.val - b.val);
    for (ListNode head : lists) {
        if (head != null) heap.offer(head);
    }

    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;
    while (!heap.isEmpty()) {
        ListNode min = heap.poll();
        curr.next = min;
        curr = curr.next;
        if (min.next != null) heap.offer(min.next);
    }
    return dummy.next;
}
```

- 时间 O(N·log K)，N 为总节点数，K 为链表数
- 空间 O(K)

**📝 解析**

- **思路推导**：每次从 K 个链表头中选最小的 → 用堆维护 K 个候选节点。取出最小的后，把它的 next 入堆。重复直到堆空。
- **关键点**：堆里最多 K 个元素（每个链表一个），所以每次操作 O(log K)。用 dummy 节点简化头节点处理。
- **易错点**：入堆前检查 `head != null`（空链表跳过）。`min.next != null` 才入堆（链表到头了就不入）。
- **举一反三**：
  - 分治法：两两合并，时间一样但常数更大
  - 合并 K 个有序数组 → 同样思路，存 [值, 数组下标, 元素下标] 三元组

## 5. 任务调度器（#621）

> 相同任务之间必须间隔 n 个冷却时间，求最短总时间

**思路**：贪心 + 堆，每次执行频率最高的任务

```java
public int leastInterval(char[] tasks, int n) {
    // 统计频率
    int[] freq = new int[26];
    for (char t : tasks) freq[t - 'A']++;

    // 大顶堆
    PriorityQueue<Integer> heap = new PriorityQueue<>(Collections.reverseOrder());
    for (int f : freq) if (f > 0) heap.offer(f);

    int time = 0;
    while (!heap.isEmpty()) {
        List<Integer> temp = new ArrayList<>();
        for (int i = 0; i <= n; i++) {
            if (!heap.isEmpty()) temp.add(heap.poll());
        }
        for (int f : temp) {
            if (--f > 0) heap.offer(f);
        }
        // 如果堆空了，最后一轮不需要等冷却
        time += heap.isEmpty() ? temp.size() : n + 1;
    }
    return time;
}
```

- 时间 O(n·26·log 26) ≈ O(n)，因为最多 26 种任务
- 贪心：优先执行频率最高的

**📝 解析**

- **思路推导**：直觉上，频率最高的任务应该最先执行，因为它决定了最少需要多少个冷却间隔。用大顶堆每次取频率最高的 n+1 个任务执行一轮。
- **关键点**：最后一轮如果堆空了，说明不需要等冷却，只加实际执行的任务数 `temp.size()`。否则加 `n+1`（完整的冷却周期）。
- **易错点**：`--f > 0` 才放回堆（频率减到 0 就不放了）。每轮最多执行 n+1 个不同的任务。
- **举一反三**：也有数学公式解法：`(maxFreq - 1) * (n + 1) + countOfMaxFreq`，面试可以先说贪心再提公式

## 6. 有序矩阵中第 K 小的元素（#378）

> n×n 矩阵，每行每列都升序，找第 K 小

**思路**：小顶堆，初始入每行第一个元素，弹出 K 次

```java
public int kthSmallest(int[][] matrix, int k) {
    int n = matrix.length;
    // 堆元素：[值, 行号, 列号]
    PriorityQueue<int[]> heap = new PriorityQueue<>((a, b) -> a[0] - b[0]);
    for (int i = 0; i < n; i++) heap.offer(new int[]{matrix[i][0], i, 0});

    for (int i = 0; i < k - 1; i++) {
        int[] curr = heap.poll();
        int col = curr[2] + 1;
        if (col < n) heap.offer(new int[]{matrix[curr[1]][col], curr[1], col});
    }
    return heap.poll()[0];
}
```

- 时间 O(k·log n)，空间 O(n)
- 也可以二分解（时间 O(n·log(max-min))）

**📝 解析**

- **思路推导**：矩阵已排序 → 每行是有序数组 → 问题变成"合并 n 个有序数组取第 K 个" → 小顶堆。初始把每行第一个元素入堆，每次弹出最小的，把该行下一个元素入堆。
- **关键点**：堆里存 [值, 行号, 列号] 三元组，弹出后根据行号和列号找到下一个元素。堆大小始终为 n。
- **易错点**：弹出后 `col + 1 < n` 才入堆（该行用完了就不入）。循环 K-1 次后堆顶就是第 K 小。
- **举一反三**：二分解法——答案在 `[matrix[0][0], matrix[n-1][n-1]]` 之间，对每个 mid 统计 ≤ mid 的元素个数，O(n·log(max-min))。
