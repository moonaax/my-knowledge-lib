# 动态规划

## 核心技巧

- DP 五步法：定义状态 → 状态转移方程 → 初始化 → 遍历顺序 → 返回值
- 常见类型：线性 DP、背包、区间 DP、树形 DP、状态压缩
- 空间优化：二维降一维（滚动数组）
- 想不出转移方程就先写暴力递归，加记忆化，再改成递推

## 1. 爬楼梯（#70）

```java
public int climbStairs(int n) {
    if (n <= 2) return n;
    int a = 1, b = 2;
    for (int i = 3; i <= n; i++) {
        int tmp = a + b;
        a = b;
        b = tmp;
    }
    return b;
}
```

- `dp[i] = dp[i-1] + dp[i-2]`，本质是斐波那契
- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：到第 i 阶可以从 i-1 跨一步或 i-2 跨两步来，所以 `dp[i] = dp[i-1] + dp[i-2]`。只依赖前两个值，滚动变量优化到 O(1) 空间。
- **关键点**：这是 DP 入门题，理解"状态定义"和"转移方程"的基础。
- **易错点**：n=1 和 n=2 要特判，否则循环不执行。
- **举一反三**：使用最小花费爬楼梯（#746）→ 加上每阶的代价；解码方法（#91）→ 类似的两步选择

## 2. 打家劫舍（#198）

> 不能偷相邻的房子，求最大金额

```java
public int rob(int[] nums) {
    int prev = 0, curr = 0;
    for (int n : nums) {
        int tmp = Math.max(curr, prev + n);
        prev = curr;
        curr = tmp;
    }
    return curr;
}
```

- `dp[i] = max(dp[i-1], dp[i-2] + nums[i])`
- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：对每间房子，选择偷（加上 i-2 的最优）或不偷（保持 i-1 的最优）。两个滚动变量 prev/curr 即可。
- **关键点**：不是"隔一个偷一个"，而是"不能偷相邻的"，可能隔两个偷。
- **易错点**：初始化 prev=0, curr=0，第一轮 curr 会变成 nums[0]。
- **举一反三**：打家劫舍 II（#213）→ 环形，分两次 DP（去头/去尾）；打家劫舍 III（#337）→ 树形 DP

## 3. 最长递增子序列（#300）

```java
public int lengthOfLIS(int[] nums) {
    int[] tails = new int[nums.length];
    int size = 0;
    for (int n : nums) {
        int lo = 0, hi = size;
        while (lo < hi) {
            int mid = (lo + hi) / 2;
            if (tails[mid] < n) lo = mid + 1;
            else hi = mid;
        }
        tails[lo] = n;
        if (lo == size) size++;
    }
    return size;
}
```

- 贪心 + 二分：tails[i] 表示长度为 i+1 的递增子序列的最小末尾
- 时间 O(n·log n)，空间 O(n)
- 普通 DP 是 O(n²)，面试优先写这个

**📝 解析**

- **思路推导**：O(n²) DP：`dp[i] = max(dp[j]+1)` 对所有 j<i 且 nums[j]<nums[i]。贪心+二分 O(n·log n)：维护 tails 数组，tails[i] 是长度为 i+1 的 LIS 的最小末尾，用二分找插入位置。
- **关键点**：tails 数组始终有序，所以能用二分。它不是实际的 LIS，只是用来计算长度。
- **易错点**：二分找的是第一个 ≥ n 的位置（lower_bound），不是 >。
- **举一反三**：最长递增子序列的个数（#673）→ 需要额外维护计数数组；俄罗斯套娃信封（#354）→ 二维 LIS

## 4. 零钱兑换（#322）

> 用最少硬币凑出 amount

```java
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1);
    dp[0] = 0;
    for (int i = 1; i <= amount; i++) {
        for (int c : coins) {
            if (c <= i) dp[i] = Math.min(dp[i], dp[i - c] + 1);
        }
    }
    return dp[amount] > amount ? -1 : dp[amount];
}
```

- 完全背包问题
- 时间 O(amount × coins)，空间 O(amount)

**📝 解析**

- **思路推导**：完全背包：每种硬币可以用无限次。`dp[i]` = 凑出金额 i 的最少硬币数。对每个金额尝试每种硬币。
- **关键点**：初始化 `dp[0]=0`，其余为 `amount+1`（相当于无穷大）。最后判断 `dp[amount] > amount` 说明无解。
- **易错点**：不能初始化为 `Integer.MAX_VALUE`，否则 +1 会溢出。
- **举一反三**：零钱兑换 II（#518）→ 求组合数而非最少个数，注意遍历顺序（先物品后金额避免重复计数）

## 5. 最长公共子序列（#1143）

```java
public int longestCommonSubsequence(String text1, String text2) {
    int m = text1.length(), n = text2.length();
    int[] dp = new int[n + 1];
    for (int i = 1; i <= m; i++) {
        int prev = 0;
        for (int j = 1; j <= n; j++) {
            int tmp = dp[j];
            if (text1.charAt(i - 1) == text2.charAt(j - 1)) dp[j] = prev + 1;
            else dp[j] = Math.max(dp[j], dp[j - 1]);
            prev = tmp;
        }
    }
    return dp[n];
}
```

- 时间 O(m×n)，空间 O(n)
- 经典二维 DP 压缩到一维

**📝 解析**

- **思路推导**：`dp[i][j]` = text1 前 i 个和 text2 前 j 个的 LCS 长度。相等时 `dp[i-1][j-1]+1`，不等时 `max(dp[i-1][j], dp[i][j-1])`。一维优化用 prev 保存左上角的值。
- **关键点**：一维优化时 `prev = tmp` 保存的是更新前的 `dp[j]`，即二维中的 `dp[i-1][j-1]`。
- **易错点**：一维优化容易搞混 prev 的更新时机，建议先写二维再优化。
- **举一反三**：最长公共子串→ 不等时 dp 归零；编辑距离（#72）→ LCS 的扩展

## 6. 编辑距离（#72）

> 把 word1 变成 word2 的最少操作数（插入、删除、替换）

```java
public int minDistance(String word1, String word2) {
    int m = word1.length(), n = word2.length();
    int[] dp = new int[n + 1];
    for (int j = 0; j <= n; j++) dp[j] = j;
    for (int i = 1; i <= m; i++) {
        int prev = dp[0];
        dp[0] = i;
        for (int j = 1; j <= n; j++) {
            int tmp = dp[j];
            if (word1.charAt(i - 1) == word2.charAt(j - 1)) dp[j] = prev;
            else dp[j] = 1 + Math.min(prev, Math.min(dp[j], dp[j - 1]));
            prev = tmp;
        }
    }
    return dp[n];
}
```

- 时间 O(m×n)，空间 O(n)
- 面试高频，必须能写

**📝 解析**

- **思路推导**：`dp[i][j]` = word1 前 i 个变成 word2 前 j 个的最少操作。相等时继承 `dp[i-1][j-1]`，不等时取三种操作的最小值+1（插入/删除/替换对应 dp 的三个方向）。
- **关键点**：`dp[i][j-1]+1` 是插入，`dp[i-1][j]+1` 是删除，`dp[i-1][j-1]+1` 是替换。
- **易错点**：初始化第一行第一列：`dp[0][j]=j`（空串变成 j 个字符需要 j 次插入）。
- **举一反三**：最长公共子序列（#1143）→ 编辑距离的简化版；正则表达式匹配（#10）→ 更复杂的状态转移

## 7. 最大子数组和（#53）

```java
public int maxSubArray(int[] nums) {
    int max = nums[0], cur = nums[0];
    for (int i = 1; i < nums.length; i++) {
        cur = Math.max(nums[i], cur + nums[i]);
        max = Math.max(max, cur);
    }
    return max;
}
```

- `dp[i] = max(nums[i], dp[i-1] + nums[i])`
- 时间 O(n)，空间 O(1)
- 也叫 Kadane 算法

**📝 解析**

- **思路推导**：`dp[i]` = 以 nums[i] 结尾的最大子数组和。要么接上前面（`dp[i-1]+nums[i]`），要么自己重新开始（`nums[i]`）。
- **关键点**：Kadane 算法本质就是这个 DP，只用一个变量 cur 滚动。全局 max 记录历史最大值。
- **易错点**：全是负数时答案是最大的那个负数，不是 0。所以 max 初始化为 `nums[0]` 不是 0。
- **举一反三**：乘积最大子数组（#152）→ 需要同时维护最大和最小；环形子数组最大和（#918）→ 考虑跨越首尾

## 8. 不同路径（#62）

> 从左上到右下有多少条路径

```java
public int uniquePaths(int m, int n) {
    int[] dp = new int[n];
    Arrays.fill(dp, 1);
    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            dp[j] += dp[j - 1];
    return dp[n - 1];
}
```

- 时间 O(m×n)，空间 O(n)

**📝 解析**

- **思路推导**：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`，只能向右或向下走。第一行和第一列都是 1。一维优化：`dp[j] += dp[j-1]`。
- **关键点**：也可以用组合数学 C(m+n-2, m-1) 直接算，但 DP 更通用。
- **易错点**：一维数组初始化全 1，代表第一行。
- **举一反三**：不同路径 II（#63）→ 有障碍物，障碍处 dp=0；最小路径和（#64）→ 求最小代价

## 9. 最小路径和（#64）

```java
public int minPathSum(int[][] grid) {
    int n = grid[0].length;
    int[] dp = new int[n];
    dp[0] = grid[0][0];
    for (int j = 1; j < n; j++) dp[j] = dp[j - 1] + grid[0][j];
    for (int i = 1; i < grid.length; i++) {
        dp[0] += grid[i][0];
        for (int j = 1; j < n; j++)
            dp[j] = grid[i][j] + Math.min(dp[j], dp[j - 1]);
    }
    return dp[n - 1];
}
```

- 时间 O(m×n)，空间 O(n)

**📝 解析**

- **思路推导**：和不同路径一样的网格 DP，`dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])`。
- **关键点**：第一行只能从左来，第一列只能从上来，单独初始化。
- **易错点**：不能贪心（每步选最小），因为局部最优不等于全局最优。
- **举一反三**：三角形最小路径和（#120）→ 自底向上 DP 更简洁

## 10. 单词拆分（#139）

> 判断字符串能否被字典中的单词拆分

```java
public boolean wordBreak(String s, List<String> wordDict) {
    Set<String> set = new HashSet<>(wordDict);
    boolean[] dp = new boolean[s.length() + 1];
    dp[0] = true;
    for (int i = 1; i <= s.length(); i++) {
        for (int j = 0; j < i; j++) {
            if (dp[j] && set.contains(s.substring(j, i))) {
                dp[i] = true;
                break;
            }
        }
    }
    return dp[s.length()];
}
```

- 时间 O(n²)，空间 O(n)

**📝 解析**

- **思路推导**：`dp[i]` = s 的前 i 个字符能否被拆分。枚举分割点 j，如果 `dp[j]=true` 且 `s[j..i]` 在字典中，则 `dp[i]=true`。
- **关键点**：用 HashSet 存字典，O(1) 查找。找到一个可行的 j 就 break。
- **易错点**：`substring(j, i)` 是左闭右开。可以优化：只枚举字典中单词长度范围内的 j。
- **举一反三**：单词拆分 II（#140）→ 返回所有拆分方案，回溯+记忆化

## 11. 最长回文子串（#5）

**思路**：中心扩展

```java
public String longestPalindrome(String s) {
    int start = 0, maxLen = 0;
    for (int i = 0; i < s.length(); i++) {
        int len1 = expand(s, i, i);     // 奇数
        int len2 = expand(s, i, i + 1); // 偶数
        int len = Math.max(len1, len2);
        if (len > maxLen) {
            maxLen = len;
            start = i - (len - 1) / 2;
        }
    }
    return s.substring(start, start + maxLen);
}

private int expand(String s, int l, int r) {
    while (l >= 0 && r < s.length() && s.charAt(l) == s.charAt(r)) { l--; r++; }
    return r - l - 1;
}
```

- 时间 O(n²)，空间 O(1)
- 比 DP 更直观，面试推荐这个写法

**📝 解析**

- **思路推导**：DP 用 `dp[i][j]` 表示 s[i..j] 是否回文，O(n²) 时间和空间。中心扩展更直观：以每个位置为中心向两边扩展，分奇偶两种情况。
- **关键点**：`start = i - (len-1)/2` 计算起始位置，统一处理奇偶。
- **易错点**：Manacher 算法可以 O(n)，但面试中中心扩展就够了。
- **举一反三**：回文子串个数（#647）→ 同样的中心扩展，计数而非找最长

## 12. 0-1 背包（模板）

> 每个物品只能选一次，求最大价值

```java
public int knapsack(int[] weights, int[] values, int capacity) {
    int[] dp = new int[capacity + 1];
    for (int i = 0; i < weights.length; i++) {
        for (int j = capacity; j >= weights[i]; j--) { // 逆序！
            dp[j] = Math.max(dp[j], dp[j - weights[i]] + values[i]);
        }
    }
    return dp[capacity];
}
```

- 逆序遍历保证每个物品只用一次
- 完全背包改为正序遍历

**📝 解析**

- **思路推导**：二维 `dp[i][j]` = 前 i 个物品、容量 j 的最大价值。一维优化后逆序遍历 j，保证每个物品只被选一次（因为 `dp[j-w]` 还是上一轮的值）。
- **关键点**：0-1 背包逆序，完全背包正序——这是最核心的区别，必须理解为什么。
- **易错点**：正序遍历时 `dp[j-w]` 可能已经包含了当前物品，相当于重复选择。
- **举一反三**：分割等和子集（#416）→ 0-1 背包变体；零钱兑换（#322）→ 完全背包

## 13. 分割等和子集（#416）

> 能否将数组分成两个和相等的子集

```java
public boolean canPartition(int[] nums) {
    int sum = 0;
    for (int n : nums) sum += n;
    if (sum % 2 != 0) return false;
    int target = sum / 2;
    boolean[] dp = new boolean[target + 1];
    dp[0] = true;
    for (int n : nums) {
        for (int j = target; j >= n; j--) {
            dp[j] = dp[j] || dp[j - n];
        }
    }
    return dp[target];
}
```

- 转化为 0-1 背包：能否凑出 sum/2
- 时间 O(n × sum)，空间 O(sum)

**📝 解析**

- **思路推导**：等和分割 = 能否从数组中选出子集使和为 sum/2。这就是 0-1 背包的"能否恰好装满"版本。
- **关键点**：sum 为奇数直接返回 false。`dp[j] = dp[j] || dp[j-n]`，选或不选当前数。
- **易错点**：逆序遍历 j，保证每个数只用一次。
- **举一反三**：目标和（#494）→ 转化为背包：选正号的和为 (sum+target)/2

## 14. 最长有效括号（#32）

```java
public int longestValidParentheses(String s) {
    int[] dp = new int[s.length()];
    int max = 0;
    for (int i = 1; i < s.length(); i++) {
        if (s.charAt(i) == ')') {
            if (s.charAt(i - 1) == '(') {
                dp[i] = (i >= 2 ? dp[i - 2] : 0) + 2;
            } else if (i - dp[i - 1] > 0 && s.charAt(i - dp[i - 1] - 1) == '(') {
                dp[i] = dp[i - 1] + 2 + (i - dp[i - 1] >= 2 ? dp[i - dp[i - 1] - 2] : 0);
            }
            max = Math.max(max, dp[i]);
        }
    }
    return max;
}
```

- 时间 O(n)，空间 O(n)

**📝 解析**

- **思路推导**：`dp[i]` = 以 s[i] 结尾的最长有效括号长度。只有 s[i]=')' 时才可能有值。分两种情况：前一个是 '(' 直接配对；前一个是 ')' 则跳过 dp[i-1] 个字符看是否有匹配的 '('。
- **关键点**：第二种情况还要加上匹配的 '(' 前面的有效长度 `dp[i-dp[i-1]-2]`。
- **易错点**：下标越界检查很多，容易写错。也可以用栈解法（栈底放最后一个未匹配的下标），更直观。
- **举一反三**：有效的括号（#20）→ 简化版，只判断是否有效

## 15. 买卖股票的最佳时机（#121）

```java
public int maxProfit(int[] prices) {
    int minPrice = Integer.MAX_VALUE, maxProfit = 0;
    for (int p : prices) {
        minPrice = Math.min(minPrice, p);
        maxProfit = Math.max(maxProfit, p - minPrice);
    }
    return maxProfit;
}
```

- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：只能买卖一次。维护历史最低价，每天算"今天卖出的利润"= 今天价格 - 历史最低价，取最大值。
- **关键点**：一次遍历同时更新 minPrice 和 maxProfit。
- **易错点**：先更新 minPrice 再算 maxProfit，或者反过来都行，因为当天买当天卖利润为 0。
- **举一反三**：买卖股票系列是经典 DP 专题——II 多次交易、III 最多两次、IV 最多 k 次、含冷冻期、含手续费

## 16. 买卖股票的最佳时机 II（#122）

> 可以多次买卖

```java
public int maxProfit(int[] prices) {
    int profit = 0;
    for (int i = 1; i < prices.length; i++) {
        if (prices[i] > prices[i - 1]) profit += prices[i] - prices[i - 1];
    }
    return profit;
}
```

- 贪心：只要涨就卖

**📝 解析**

- **思路推导**：可以多次买卖。贪心：所有上涨段的利润之和就是最大利润。等价于每天决定是否持有，DP 也能做但贪心更简洁。
- **关键点**：`prices[i] - prices[i-1] > 0` 就累加，相当于在每个上涨日都"买入前一天卖出今天"。
- **易错点**：不是找波峰波谷，而是简单地累加所有正差值，结果一样但代码更简单。
- **举一反三**：含冷冻期（#309）→ 卖出后隔一天才能买，需要状态机 DP

## 17. 乘积最大子数组（#152）

```java
public int maxProduct(int[] nums) {
    int max = nums[0], curMax = nums[0], curMin = nums[0];
    for (int i = 1; i < nums.length; i++) {
        if (nums[i] < 0) { int tmp = curMax; curMax = curMin; curMin = tmp; }
        curMax = Math.max(nums[i], curMax * nums[i]);
        curMin = Math.min(nums[i], curMin * nums[i]);
        max = Math.max(max, curMax);
    }
    return max;
}
```

- 同时维护最大和最小（负负得正）
- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：和最大子数组和不同，乘积有负数翻转的特性。负数 × 最小值可能变成最大值，所以要同时维护 curMax 和 curMin。
- **关键点**：遇到负数时先交换 curMax 和 curMin，再乘以 nums[i]。
- **易错点**：不能只维护 curMax，`[-2,3,-4]` 中 -2×3×-4=24 是最大值，需要 curMin 传递负数。
- **举一反三**：最大子数组和（#53）→ 只有加法，不需要维护最小值

## 18. 正则表达式匹配（#10）

```java
public boolean isMatch(String s, String p) {
    int m = s.length(), n = p.length();
    boolean[][] dp = new boolean[m + 1][n + 1];
    dp[0][0] = true;
    for (int j = 2; j <= n; j++)
        if (p.charAt(j - 1) == '*') dp[0][j] = dp[0][j - 2];
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (p.charAt(j - 1) == '*') {
                dp[i][j] = dp[i][j - 2]; // * 匹配 0 次
                if (p.charAt(j - 2) == '.' || p.charAt(j - 2) == s.charAt(i - 1))
                    dp[i][j] |= dp[i - 1][j]; // * 匹配 1+ 次
            } else if (p.charAt(j - 1) == '.' || p.charAt(j - 1) == s.charAt(i - 1)) {
                dp[i][j] = dp[i - 1][j - 1];
            }
        }
    }
    return dp[m][n];
}
```

- 时间 O(m×n)，空间 O(m×n)

**📝 解析**

- **思路推导**：`dp[i][j]` = s 前 i 个和 p 前 j 个是否匹配。关键是处理 `*`：匹配 0 次（`dp[i][j-2]`）或匹配 1+ 次（`dp[i-1][j]` 且当前字符匹配）。
- **关键点**：`*` 不能单独出现，它和前一个字符组合。`p[j-1]=='*'` 时看 `p[j-2]` 是否匹配 `s[i-1]`。
- **易错点**：初始化 `dp[0][j]`：`a*b*` 这种模式可以匹配空串，需要处理。
- **举一反三**：通配符匹配（#44）→ `?` 匹配单个字符，`*` 匹配任意序列，比正则简单

## 19. 不同的二叉搜索树（#96）

```java
public int numTrees(int n) {
    int[] dp = new int[n + 1];
    dp[0] = dp[1] = 1;
    for (int i = 2; i <= n; i++)
        for (int j = 1; j <= i; j++)
            dp[i] += dp[j - 1] * dp[i - j];
    return dp[n];
}
```

- 卡特兰数：`G(n) = Σ G(i-1) × G(n-i)`
- 时间 O(n²)，空间 O(n)

**📝 解析**

- **思路推导**：以 j 为根时，左子树有 j-1 个节点，右子树有 n-j 个节点。总数 = Σ 左子树种数 × 右子树种数。这就是卡特兰数的递推公式。
- **关键点**：`dp[0]=1` 表示空树也是一种 BST。卡特兰数还有直接公式 `C(2n,n)/(n+1)`。
- **易错点**：内层循环 j 从 1 到 i，不是从 0 开始。
- **举一反三**：不同的二叉搜索树 II（#95）→ 返回所有具体的树结构，用递归构造

## 20. 解码方法（#91）

> "226" 可以解码为 "BZ"(2,26)、"VF"(22,6)、"BBF"(2,2,6)

```java
public int numDecodings(String s) {
    if (s.charAt(0) == '0') return 0;
    int prev2 = 1, prev1 = 1;
    for (int i = 1; i < s.length(); i++) {
        int curr = 0;
        if (s.charAt(i) != '0') curr = prev1;
        int two = Integer.parseInt(s.substring(i - 1, i + 1));
        if (two >= 10 && two <= 26) curr += prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

- 时间 O(n)，空间 O(1)
- 踩坑：'0' 不能单独解码

**📝 解析**

- **思路推导**：类似爬楼梯，每次可以解码 1 位或 2 位。`dp[i]` = 前 i 个字符的解码方法数。单独解码 s[i]（非 '0' 时加 dp[i-1]），两位解码 s[i-1..i]（10~26 时加 dp[i-2]）。
- **关键点**：'0' 不能单独解码，所以 s[i]='0' 时不加 dp[i-1]。两位数必须在 10~26 范围内。
- **易错点**：`"06"` 无法解码返回 0；`"10"` 只有一种解码。开头是 '0' 直接返回 0。
- **举一反三**：爬楼梯（#70）→ 去掉 '0' 的限制就是爬楼梯；解码方法 II（#639）→ 加入 `*` 通配符
