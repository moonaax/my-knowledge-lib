# 链表

## 核心技巧

- 虚拟头节点（dummy）：统一处理头节点的增删
- 快慢指针：找中点、判环、找环入口
- 递归：天然适合链表的后序处理（反转、合并）
- 画图：链表题一定要画指针变化过程

---

## 1. 反转链表（#206）

```java
public ListNode reverseList(ListNode head) {
    ListNode prev = null, curr = head;
    while (curr != null) {
        ListNode next = curr.next;
        curr.next = prev;
        prev = curr;
        curr = next;
    }
    return prev;
}
```

- 时间 O(n)，空间 O(1)
- 必须先保存 next，再断链

**📝 解析**

- **思路推导**：三个指针 prev/curr/next，每次把 curr.next 指向 prev，然后三个指针各前进一步。
- **关键点**：先 `next = curr.next` 保存后继，否则断链后就丢了。
- **易错点**：递归写法更简洁但空间 O(n)，面试优先写迭代。
- **举一反三**：反转链表 II（#92）→ 反转指定区间；K 个一组翻转链表（#25）→ 分段反转

## 2. 合并两个有序链表（#21）

```java
public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0), curr = dummy;
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) { curr.next = l1; l1 = l1.next; }
        else { curr.next = l2; l2 = l2.next; }
        curr = curr.next;
    }
    curr.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}
```

- 时间 O(m+n)，空间 O(1)

**📝 解析**

- **思路推导**：dummy 节点统一处理，避免单独判断头节点。每次取较小的接到 curr 后面。
- **关键点**：最后 `curr.next = l1 != null ? l1 : l2` 把剩余部分直接接上。
- **易错点**：别忘了返回 `dummy.next` 而不是 dummy。
- **举一反三**：合并 K 个升序链表（#23）→ 用堆或分治扩展

## 3. 环形链表（#141）

> 判断链表是否有环

```java
public boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true;
    }
    return false;
}
```

- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：HashSet 存访问过的节点 O(n) 空间。快慢指针 O(1) 空间：快走两步慢走一步，有环必相遇。
- **关键点**：判断条件是 `fast != null && fast.next != null`，保证 fast 能走两步。
- **易错点**：不能用 `fast.next != null && fast.next.next != null`，会漏掉 fast 本身为 null 的情况。
- **举一反三**：环形链表 II（#142）→ 找环入口，相遇后再走一轮

## 4. 环形链表 II（#142）

> 找环的入口节点

**思路**：快慢指针相遇后，一个从 head 出发，一个从相遇点出发，再次相遇即入口

```java
public ListNode detectCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) {
            ListNode p = head;
            while (p != slow) {
                p = p.next;
                slow = slow.next;
            }
            return p;
        }
    }
    return null;
}
```

- 数学证明：设头到入口 a，入口到相遇点 b，环长 c，则 a = c - b

**📝 解析**

- **思路推导**：快慢指针相遇时，slow 走了 a+b，fast 走了 a+b+nc。因为 fast=2×slow，所以 a+b=nc，即 a=nc-b=(n-1)c+(c-b)。从 head 和相遇点同时走，走 a 步后在入口相遇。
- **关键点**：数学推导是核心，面试时要能讲清楚为什么第二次相遇就是入口。
- **易错点**：第二阶段两个指针都走一步，不是一快一慢。
- **举一反三**：找重复数（#287）→ 把数组值当 next 指针，转化为找环入口

## 5. 删除链表的倒数第 N 个节点（#19）

**思路**：快指针先走 n 步

```java
public ListNode removeNthFromEnd(ListNode head, int n) {
    ListNode dummy = new ListNode(0, head);
    ListNode fast = dummy, slow = dummy;
    for (int i = 0; i <= n; i++) fast = fast.next;
    while (fast != null) {
        fast = fast.next;
        slow = slow.next;
    }
    slow.next = slow.next.next;
    return dummy.next;
}
```

- 时间 O(n)，空间 O(1)
- 踩坑：用 dummy 处理删除头节点的情况

**📝 解析**

- **思路推导**：两趟遍历（先算长度再定位）或一趟（快指针先走 n 步）。快指针先走 n+1 步，这样 slow 停在待删节点的前一个。
- **关键点**：dummy 节点让 fast 多走一步（循环 `i <= n`），slow 最终指向待删节点的前驱。
- **易错点**：删除头节点时没有 dummy 会出错，比如 `[1] n=1`。
- **举一反三**：链表的中间节点（#876）→ 快慢指针找中点，快走两步慢走一步

## 6. 两数相加（#2）

> 两个链表表示的数相加

```java
public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0), curr = dummy;
    int carry = 0;
    while (l1 != null || l2 != null || carry > 0) {
        int sum = carry;
        if (l1 != null) { sum += l1.val; l1 = l1.next; }
        if (l2 != null) { sum += l2.val; l2 = l2.next; }
        curr.next = new ListNode(sum % 10);
        carry = sum / 10;
        curr = curr.next;
    }
    return dummy.next;
}
```

- 时间 O(max(m,n))，空间 O(1)

**📝 解析**

- **思路推导**：链表已经是逆序存储，直接从头开始加，天然对齐个位。逐位相加，处理进位。
- **关键点**：循环条件 `l1 != null || l2 != null || carry > 0`，三个条件缺一不可，最后的进位容易漏。
- **易错点**：两个链表长度不同时，短的那个要当 0 处理。
- **举一反三**：两数相加 II（#445）→ 高位在前，需要先反转或用栈

## 7. 回文链表（#234）

**思路**：快慢指针找中点 → 反转后半段 → 比较

```java
public boolean isPalindrome(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    ListNode rev = reverse(slow);
    while (rev != null) {
        if (head.val != rev.val) return false;
        head = head.next;
        rev = rev.next;
    }
    return true;
}

private ListNode reverse(ListNode head) {
    ListNode prev = null;
    while (head != null) {
        ListNode next = head.next;
        head.next = prev;
        prev = head;
        head = next;
    }
    return prev;
}
```

- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：转数组再判断 O(n) 空间。O(1) 空间：快慢指针找中点 → 反转后半段 → 逐一比较。
- **关键点**：奇数长度时 slow 停在正中间，反转 slow 开始的后半段，前半段多一个节点不影响比较。
- **易错点**：如果要求不修改链表，比较完后需要把后半段再反转回去。
- **举一反三**：组合了找中点（#876）+ 反转链表（#206）两个子问题

## 8. 合并 K 个升序链表（#23）

**思路**：小顶堆

```java
public ListNode mergeKLists(ListNode[] lists) {
    PriorityQueue<ListNode> pq = new PriorityQueue<>((a, b) -> a.val - b.val);
    for (ListNode l : lists) if (l != null) pq.offer(l);
    ListNode dummy = new ListNode(0), curr = dummy;
    while (!pq.isEmpty()) {
        ListNode node = pq.poll();
        curr.next = node;
        curr = curr.next;
        if (node.next != null) pq.offer(node.next);
    }
    return dummy.next;
}
```

- 时间 O(n·log k)，k 为链表数量
- 也可用分治合并，复杂度相同

**📝 解析**

- **思路推导**：逐一合并 O(nk)。小顶堆维护 k 个链表的当前头节点，每次取最小的 O(log k)，总 O(n·log k)。分治两两合并也是 O(n·log k)。
- **关键点**：堆里存节点引用，弹出后把 node.next 入堆。入堆前判空。
- **易错点**：Java 的 PriorityQueue 比较器 `(a,b)->a.val-b.val`，极端值可能溢出，用 `Integer.compare` 更安全。
- **举一反三**：外部排序的核心思想就是多路归并

## 9. 排序链表（#148）

**思路**：归并排序（找中点 → 分割 → 合并）

```java
public ListNode sortList(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode slow = head, fast = head.next;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    ListNode mid = slow.next;
    slow.next = null;
    return mergeTwoLists(sortList(head), sortList(mid));
}

private ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0), curr = dummy;
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) { curr.next = l1; l1 = l1.next; }
        else { curr.next = l2; l2 = l2.next; }
        curr = curr.next;
    }
    curr.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}
```

- 时间 O(n·log n)，空间 O(log n) 递归栈
- 踩坑：找中点时 fast 从 head.next 开始，保证偶数长度时 slow 在左半段末尾

**📝 解析**

- **思路推导**：链表不能随机访问，快排不适合。归并排序天然适合：找中点断开 → 递归排序两半 → 合并。
- **关键点**：fast 从 `head.next` 开始而不是 head，这样偶数长度时 slow 停在左半段最后一个节点，方便断开。
- **易错点**：断开链表 `slow.next = null` 必须在递归之前，否则死循环。
- **举一反三**：自底向上归并可以做到 O(1) 空间，但代码复杂很多

## 10. LRU 缓存（#146）

**思路**：HashMap + 双向链表

```java
class LRUCache {
    private Map<Integer, Node> map = new HashMap<>();
    private Node head = new Node(), tail = new Node();
    private int cap;

    public LRUCache(int capacity) {
        cap = capacity;
        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        Node node = map.get(key);
        if (node == null) return -1;
        moveToHead(node);
        return node.val;
    }

    public void put(int key, int value) {
        Node node = map.get(key);
        if (node != null) {
            node.val = value;
            moveToHead(node);
        } else {
            node = new Node(key, value);
            map.put(key, node);
            addToHead(node);
            if (map.size() > cap) {
                Node removed = tail.prev;
                removeNode(removed);
                map.remove(removed.key);
            }
        }
    }

    private void addToHead(Node node) {
        node.next = head.next;
        node.prev = head;
        head.next.prev = node;
        head.next = node;
    }

    private void removeNode(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    private void moveToHead(Node node) {
        removeNode(node);
        addToHead(node);
    }

    static class Node {
        int key, val;
        Node prev, next;
        Node() {}
        Node(int k, int v) { key = k; val = v; }
    }
}
```

- get/put 均 O(1)
- 面试超高频，必须能手写

**📝 解析**

- **思路推导**：O(1) get 需要 HashMap，O(1) 调整顺序需要双向链表。两者结合：HashMap 存 key→Node，双向链表维护访问顺序。
- **关键点**：head/tail 是哨兵节点，简化边界处理。Node 必须存 key，因为淘汰时需要从 HashMap 中删除对应 key。
- **易错点**：put 已存在的 key 时要更新 value 并移到头部，不能重复创建节点。
- **举一反三**：LFU 缓存（#460）→ 多一个频率维度，用频率→双向链表的 HashMap
