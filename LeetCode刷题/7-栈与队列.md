# 栈与队列

## 核心技巧

- 单调栈：找下一个更大/更小元素，O(n)
- 栈匹配：括号匹配、表达式求值
- 辅助栈：最小栈、用栈模拟队列

---

## 1. 有效的括号（#20）

```java
public boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(') stack.push(')');
        else if (c == '[') stack.push(']');
        else if (c == '{') stack.push('}');
        else if (stack.isEmpty() || stack.pop() != c) return false;
    }
    return stack.isEmpty();
}
```

- 技巧：遇到左括号压入对应右括号，匹配时直接比较

**📝 解析**

- **思路推导**：栈天然适合括号匹配。巧妙之处：压入的是期望的右括号，弹出时直接比较，不需要 if-else 判断配对关系。
- **关键点**：最后检查 `stack.isEmpty()`，处理多余左括号的情况如 `"(("`。
- **易错点**：弹出前要检查栈是否为空，否则 `")("` 会抛异常。
- **举一反三**：最长有效括号（#32）→ 用栈记录下标求长度

## 2. 最小栈（#155）

```java
class MinStack {
    private Deque<int[]> stack = new ArrayDeque<>();

    public void push(int val) {
        int min = stack.isEmpty() ? val : Math.min(val, stack.peek()[1]);
        stack.push(new int[]{val, min});
    }

    public void pop() { stack.pop(); }
    public int top() { return stack.peek()[0]; }
    public int getMin() { return stack.peek()[1]; }
}
```

- 每个元素存 [值, 当前最小值]，O(1) 获取最小值

**📝 解析**

- **思路推导**：普通栈 getMin 需要 O(n) 遍历。用辅助栈或元组存每个状态下的最小值，空间换时间。
- **关键点**：每次 push 时计算 `min(val, 当前栈顶最小值)`，和 val 一起存入。pop 时最小值自动恢复。
- **易错点**：也可以用两个栈（主栈+最小栈），最小栈只在新最小值时压入，但要处理相等的情况。
- **举一反三**：最大栈→同理维护最大值；用队列实现栈（#225）→ 两个队列互倒

## 3. 每日温度（#739）

> 找每天之后第一个更高温度的天数

**思路**：单调递减栈

```java
public int[] dailyTemperatures(int[] temperatures) {
    int n = temperatures.length;
    int[] res = new int[n];
    Deque<Integer> stack = new ArrayDeque<>(); // 存下标
    for (int i = 0; i < n; i++) {
        while (!stack.isEmpty() && temperatures[i] > temperatures[stack.peek()]) {
            int idx = stack.pop();
            res[idx] = i - idx;
        }
        stack.push(i);
    }
    return res;
}
```

- 时间 O(n)，空间 O(n)
- 单调栈模板：栈里存下标，遇到更大的就弹出计算

**📝 解析**

- **思路推导**：暴力对每天向后找 O(n²)。单调递减栈：栈里存还没找到更高温度的天的下标。遇到更高温度时，弹出所有比它矮的，计算天数差。
- **关键点**：每个元素最多入栈出栈各一次，总时间 O(n)。栈里存下标而不是值，因为需要计算距离。
- **易错点**：没有更高温度的天，res 默认为 0，不需要特殊处理。
- **举一反三**：下一个更大元素（#496/#503）→ 同样的单调栈模板

## 4. 柱状图中最大的矩形（#84）

**思路**：单调递增栈，找每个柱子左右第一个更矮的位置

```java
public int largestRectangleArea(int[] heights) {
    int n = heights.length, max = 0;
    Deque<Integer> stack = new ArrayDeque<>();
    for (int i = 0; i <= n; i++) {
        int h = (i == n) ? 0 : heights[i];
        while (!stack.isEmpty() && h < heights[stack.peek()]) {
            int height = heights[stack.pop()];
            int width = stack.isEmpty() ? i : i - stack.peek() - 1;
            max = Math.max(max, height * width);
        }
        stack.push(i);
    }
    return max;
}
```

- 时间 O(n)，空间 O(n)
- 末尾加哨兵 0 确保所有元素出栈

**📝 解析**

- **思路推导**：暴力对每个柱子向两边扩展找边界 O(n²)。单调递增栈：遇到更矮的柱子时，弹出的柱子就找到了右边界，栈顶就是左边界。宽度 = `i - stack.peek() - 1`。
- **关键点**：末尾加一个高度为 0 的哨兵，确保所有柱子都能出栈被计算。栈空时宽度为 i。
- **易错点**：弹出后计算宽度时，如果栈空说明弹出的柱子是最矮的，宽度就是 i（从最左到当前位置）。
- **举一反三**：最大矩形（#85）→ 逐行转化为柱状图，调用本题的解法

## 5. 字符串解码（#394）

> "3[a2[c]]" → "accaccacc"

```java
public String decodeString(String s) {
    Deque<StringBuilder> strStack = new ArrayDeque<>();
    Deque<Integer> numStack = new ArrayDeque<>();
    StringBuilder curr = new StringBuilder();
    int num = 0;
    for (char c : s.toCharArray()) {
        if (Character.isDigit(c)) {
            num = num * 10 + (c - '0');
        } else if (c == '[') {
            numStack.push(num);
            strStack.push(curr);
            curr = new StringBuilder();
            num = 0;
        } else if (c == ']') {
            int repeat = numStack.pop();
            StringBuilder tmp = strStack.pop();
            for (int i = 0; i < repeat; i++) tmp.append(curr);
            curr = tmp;
        } else {
            curr.append(c);
        }
    }
    return curr.toString();
}
```

- 时间 O(输出长度)，空间 O(嵌套深度)

**📝 解析**

- **思路推导**：遇到 `[` 把当前字符串和数字压栈，开始新的一层。遇到 `]` 弹出，把当前字符串重复 n 次拼接到上一层。
- **关键点**：两个栈分别存数字和字符串，或者用一个栈存 pair。遇到 `[` 时重置 curr 和 num。
- **易错点**：数字可能是多位数如 `100[a]`，要用 `num = num*10 + (c-'0')` 累积。
- **举一反三**：也可以用递归解，把 `[` 当作递归入口，`]` 当作返回

## 6. 下一个更大元素 I（#496）

```java
public int[] nextGreaterElement(int[] nums1, int[] nums2) {
    Map<Integer, Integer> map = new HashMap<>();
    Deque<Integer> stack = new ArrayDeque<>();
    for (int n : nums2) {
        while (!stack.isEmpty() && n > stack.peek()) {
            map.put(stack.pop(), n);
        }
        stack.push(n);
    }
    int[] res = new int[nums1.length];
    for (int i = 0; i < nums1.length; i++) {
        res[i] = map.getOrDefault(nums1[i], -1);
    }
    return res;
}
```

- 单调栈 + HashMap 映射结果

**📝 解析**

- **思路推导**：先对 nums2 用单调栈求出每个元素的下一个更大元素，存入 HashMap。然后对 nums1 直接查表。
- **关键点**：单调栈处理 nums2 是 O(n)，查询 nums1 是 O(m)，总 O(m+n)。
- **易错点**：栈里没被弹出的元素没有更大元素，`getOrDefault` 返回 -1。
- **举一反三**：下一个更大元素 II（#503）→ 循环数组，遍历两遍或取模处理

## 7. 用栈实现队列（#232）

```java
class MyQueue {
    private Deque<Integer> in = new ArrayDeque<>();
    private Deque<Integer> out = new ArrayDeque<>();

    public void push(int x) { in.push(x); }

    public int pop() { peek(); return out.pop(); }

    public int peek() {
        if (out.isEmpty()) while (!in.isEmpty()) out.push(in.pop());
        return out.peek();
    }

    public boolean empty() { return in.isEmpty() && out.isEmpty(); }
}
```

- 均摊 O(1)
- 关键：out 空了才从 in 倒过来

**📝 解析**

- **思路推导**：一个栈负责入队（in），一个负责出队（out）。out 空时把 in 全部倒入 out，顺序就反过来了，实现 FIFO。
- **关键点**：每个元素最多被搬运一次（从 in 到 out），所以均摊 O(1)。
- **易错点**：只有 out 为空时才倒，不是每次 pop/peek 都倒。否则顺序会乱。
- **举一反三**：用队列实现栈（#225）→ 每次 push 后把前面的元素重新入队到后面

## 8. 滑动窗口最大值（#239）

> 返回每个窗口的最大值

**思路**：单调递减双端队列

```java
public int[] maxSlidingWindow(int[] nums, int k) {
    int n = nums.length;
    int[] res = new int[n - k + 1];
    Deque<Integer> deque = new ArrayDeque<>(); // 存下标
    for (int i = 0; i < n; i++) {
        while (!deque.isEmpty() && deque.peekFirst() < i - k + 1) deque.pollFirst();
        while (!deque.isEmpty() && nums[deque.peekLast()] < nums[i]) deque.pollLast();
        deque.offerLast(i);
        if (i >= k - 1) res[i - k + 1] = nums[deque.peekFirst()];
    }
    return res;
}
```

- 时间 O(n)，空间 O(k)
- 队头是窗口最大值的下标，过期就移除

**📝 解析**

- **思路推导**：暴力每个窗口找最大 O(nk)。单调递减双端队列：队头始终是当前窗口最大值的下标。新元素入队前，从队尾移除所有比它小的（它们不可能再成为最大值）。
- **关键点**：队头过期（`deque.peekFirst() < i-k+1`）时从队头移除。队尾维护单调性。两端都要操作，所以用双端队列。
- **易错点**：`i >= k-1` 时才开始记录结果（窗口还没满时不记录）。
- **举一反三**：这是单调队列的经典应用，也用于 DP 优化（如多重背包的单调队列优化）
