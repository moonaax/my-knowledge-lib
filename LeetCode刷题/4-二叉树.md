# 二叉树

## 核心技巧

- 递归三要素：返回值、终止条件、单层逻辑
- 前中后序遍历：前序（根左右）、中序（左根右）、后序（左右根）
- 层序遍历：BFS + 队列
- 大部分树的题都是 DFS 递归，想清楚"当前节点该做什么"

---

## 1. 二叉树的中序遍历（#94）

```java
public List<Integer> inorderTraversal(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    Deque<TreeNode> stack = new ArrayDeque<>();
    TreeNode curr = root;
    while (curr != null || !stack.isEmpty()) {
        while (curr != null) {
            stack.push(curr);
            curr = curr.left;
        }
        curr = stack.pop();
        res.add(curr.val);
        curr = curr.right;
    }
    return res;
}
```

- 时间 O(n)，空间 O(n)
- 迭代写法面试更加分

**📝 解析**

- **思路推导**：递归写法三行代码但空间 O(h)。迭代用栈模拟：一路向左压栈，弹出时访问，再转向右子树。
- **关键点**：`curr != null || !stack.isEmpty()` 两个条件缺一不可，curr 为空但栈非空说明还有节点没处理。
- **易错点**：前序/后序的迭代写法不同，中序是最经典的模板。
- **举一反三**：前序迭代→根先入栈再右左入栈；后序→前序(根右左)再反转；Morris 遍历→O(1) 空间

## 2. 二叉树的最大深度（#104）

```java
public int maxDepth(TreeNode root) {
    if (root == null) return 0;
    return 1 + Math.max(maxDepth(root.left), maxDepth(root.right));
}
```

- 时间 O(n)，空间 O(h)

**📝 解析**

- **思路推导**：当前节点的深度 = 1 + max(左子树深度, 右子树深度)。递归到 null 返回 0。
- **关键点**：这是后序遍历思想——先算左右子树的结果，再处理当前节点。
- **易错点**：BFS 层序遍历也能求深度（层数就是深度），但递归更简洁。
- **举一反三**：二叉树的最小深度（#111）→ 注意单子树为空时不能取 0

## 3. 翻转二叉树（#226）

```java
public TreeNode invertTree(TreeNode root) {
    if (root == null) return null;
    TreeNode tmp = root.left;
    root.left = invertTree(root.right);
    root.right = invertTree(tmp);
    return root;
}
```

- 时间 O(n)，空间 O(h)

**📝 解析**

- **思路推导**：每个节点交换左右子树，递归处理。前序/后序都行，中序会出问题（左子树翻转后变成右子树又被翻转回去）。
- **关键点**：先保存 `root.left` 到 tmp，因为赋值后原 left 引用丢失。
- **易错点**：这题 Google 面试考过，Homebrew 作者没做出来成了梗。
- **举一反三**：对称二叉树（#101）→ 判断是否是自身的翻转

## 4. 对称二叉树（#101）

```java
public boolean isSymmetric(TreeNode root) {
    return check(root.left, root.right);
}

private boolean check(TreeNode l, TreeNode r) {
    if (l == null && r == null) return true;
    if (l == null || r == null) return false;
    return l.val == r.val && check(l.left, r.right) && check(l.right, r.left);
}
```

- 时间 O(n)，空间 O(h)

**📝 解析**

- **思路推导**：对称 = 左子树的左 == 右子树的右 && 左子树的右 == 右子树的左。双指针递归比较。
- **关键点**：递归函数接收两个节点参数，比较的是镜像位置。
- **易错点**：不能只比较左右子树是否相等，要比较镜像位置。
- **举一反三**：相同的树（#100）→ 比较两棵树是否完全相同，更简单

## 5. 二叉树的直径（#543）

> 任意两节点间最长路径的边数

```java
int max = 0;

public int diameterOfBinaryTree(TreeNode root) {
    depth(root);
    return max;
}

private int depth(TreeNode node) {
    if (node == null) return 0;
    int l = depth(node.left);
    int r = depth(node.right);
    max = Math.max(max, l + r);
    return 1 + Math.max(l, r);
}
```

- 时间 O(n)，空间 O(h)
- 核心：直径 = 左深度 + 右深度

**📝 解析**

- **思路推导**：直径不一定过根节点。对每个节点计算"经过它的最长路径"= 左深度+右深度，取全局最大值。
- **关键点**：函数返回的是深度（给父节点用），但过程中更新的是直径（全局变量）。这种"返回值和更新值不同"的模式很常见。
- **易错点**：直径是边数不是节点数，所以是 l+r 不是 l+r+1。
- **举一反三**：二叉树的最大路径和（#124）→ 同样的模式，把深度换成路径和

## 6. 二叉树的层序遍历（#102）

```java
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    while (!queue.isEmpty()) {
        int size = queue.size();
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);
            if (node.left != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        res.add(level);
    }
    return res;
}
```

- 时间 O(n)，空间 O(n)

**📝 解析**

- **思路推导**：BFS + 队列。关键是用 `size = queue.size()` 记录当前层节点数，循环 size 次处理完一层。
- **关键点**：每层开始前记录 size，这样新入队的子节点不会混入当前层。
- **易错点**：别忘了判断 root == null 直接返回空列表。
- **举一反三**：锯齿形层序遍历（#103）→ 奇数层反转；层序遍历的变体非常多

## 7. 验证二叉搜索树（#98）

**思路**：中序遍历递增 / 递归传上下界

```java
public boolean isValidBST(TreeNode root) {
    return validate(root, Long.MIN_VALUE, Long.MAX_VALUE);
}

private boolean validate(TreeNode node, long min, long max) {
    if (node == null) return true;
    if (node.val <= min || node.val >= max) return false;
    return validate(node.left, min, node.val) && validate(node.right, node.val, max);
}
```

- 时间 O(n)，空间 O(h)
- 踩坑：用 long 防止节点值为 Integer.MIN_VALUE

**📝 解析**

- **思路推导**：不能只比较节点和左右孩子的大小，要保证整个左子树都小于根、整个右子树都大于根。传递上下界递归。
- **关键点**：中序遍历也能验证（递增序列），但上下界法更直观。
- **易错点**：初始上下界用 `Long.MIN_VALUE/MAX_VALUE`，因为节点值可能是 `Integer.MIN_VALUE`。条件是严格小于/大于，不能等于。
- **举一反三**：恢复二叉搜索树（#99）→ 中序遍历找两个逆序节点

## 8. 二叉搜索树中第 K 小的元素（#230）

**思路**：中序遍历第 k 个

```java
public int kthSmallest(TreeNode root, int k) {
    Deque<TreeNode> stack = new ArrayDeque<>();
    TreeNode curr = root;
    while (curr != null || !stack.isEmpty()) {
        while (curr != null) {
            stack.push(curr);
            curr = curr.left;
        }
        curr = stack.pop();
        if (--k == 0) return curr.val;
        curr = curr.right;
    }
    return -1;
}
```

- 时间 O(H+k)，空间 O(H)

**📝 解析**

- **思路推导**：BST 中序遍历是升序序列，第 k 个就是答案。迭代中序遍历，计数到 k 就返回。
- **关键点**：不需要遍历完整棵树，找到第 k 个就停，时间 O(H+k)。
- **易错点**：`--k == 0` 是先减再判断，等价于"第 k 个被弹出时返回"。
- **举一反三**：如果频繁查询，可以在节点中维护左子树大小，O(H) 查询

## 9. 二叉树的最近公共祖先（#236）

```java
public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) return root;
    TreeNode left = lowestCommonAncestor(root.left, p, q);
    TreeNode right = lowestCommonAncestor(root.right, p, q);
    if (left != null && right != null) return root;
    return left != null ? left : right;
}
```

- 时间 O(n)，空间 O(h)
- 三种情况：p q 分居两侧 → 当前节点；都在左 → 左子树结果；都在右 → 右子树结果

**📝 解析**

- **思路推导**：后序遍历，从底向上。遇到 p 或 q 就返回自己，否则看左右子树的返回值。
- **关键点**：左右都非空说明 p q 分居两侧，当前节点就是 LCA。只有一边非空就返回那边。
- **易错点**：p 是 q 的祖先时，p 自己就是 LCA，代码中 `root == p` 的判断已经覆盖了。
- **举一反三**：BST 的最近公共祖先（#235）→ 利用 BST 性质，比较值大小即可，更简单

## 10. 从前序与中序遍历序列构造二叉树（#105）

```java
private Map<Integer, Integer> inMap = new HashMap<>();

public TreeNode buildTree(int[] preorder, int[] inorder) {
    for (int i = 0; i < inorder.length; i++) inMap.put(inorder[i], i);
    return build(preorder, 0, preorder.length - 1, 0);
}

private TreeNode build(int[] pre, int preL, int preR, int inL) {
    if (preL > preR) return null;
    TreeNode root = new TreeNode(pre[preL]);
    int inRoot = inMap.get(pre[preL]);
    int leftSize = inRoot - inL;
    root.left = build(pre, preL + 1, preL + leftSize, inL);
    root.right = build(pre, preL + leftSize + 1, preR, inRoot + 1);
    return root;
}
```

- 时间 O(n)，空间 O(n)
- 核心：前序第一个是根，在中序中找到根的位置，左边是左子树，右边是右子树

**📝 解析**

- **思路推导**：前序确定根，中序确定左右子树范围。用 HashMap 存中序值→下标，O(1) 查找根的位置。
- **关键点**：`leftSize = inRoot - inL` 算出左子树节点数，用它划分前序数组的左右子树范围。
- **易错点**：前序数组的左子树范围是 `[preL+1, preL+leftSize]`，右子树是 `[preL+leftSize+1, preR]`，下标容易算错。
- **举一反三**：从中序与后序构造（#106）→ 后序最后一个是根，其余逻辑类似

## 11. 二叉树展开为链表（#114）

> 原地展开为先序遍历的单链表

```java
public void flatten(TreeNode root) {
    TreeNode curr = root;
    while (curr != null) {
        if (curr.left != null) {
            TreeNode prev = curr.left;
            while (prev.right != null) prev = prev.right;
            prev.right = curr.right;
            curr.right = curr.left;
            curr.left = null;
        }
        curr = curr.right;
    }
}
```

- 时间 O(n)，空间 O(1)
- 思路：把左子树的最右节点接到右子树前面

**📝 解析**

- **思路推导**：前序遍历存到数组再重建 O(n) 空间。O(1) 空间：对每个节点，找左子树的最右节点（前序遍历中左子树的最后一个），把右子树接到它后面，再把左子树移到右边。
- **关键点**：这其实是 Morris 遍历的思想——利用空闲指针建立临时连接。
- **易错点**：处理完后 `curr.left = null`，别忘了清空左指针。
- **举一反三**：二叉搜索树转双向链表→类似的指针重连操作

## 12. 路径总和 III（#437）

> 找节点值之和等于 targetSum 的路径数（不需要从根开始）

**思路**：前缀和 + HashMap

```java
public int pathSum(TreeNode root, int targetSum) {
    Map<Long, Integer> map = new HashMap<>();
    map.put(0L, 1);
    return dfs(root, 0, targetSum, map);
}

private int dfs(TreeNode node, long sum, int target, Map<Long, Integer> map) {
    if (node == null) return 0;
    sum += node.val;
    int count = map.getOrDefault(sum - target, 0);
    map.merge(sum, 1, Integer::sum);
    count += dfs(node.left, sum, target, map);
    count += dfs(node.right, sum, target, map);
    map.merge(sum, -1, Integer::sum); // 回溯
    return count;
}
```

- 时间 O(n)，空间 O(n)
- 和数组前缀和思路一样，只是在树上做

**📝 解析**

- **思路推导**：暴力对每个节点做 DFS O(n²)。前缀和优化：和数组的和为 K 的子数组（#560）完全一样的思路，只是在树上做。
- **关键点**：回溯时 `map.merge(sum, -1, Integer::sum)` 撤销当前路径的前缀和，保证不影响其他分支。
- **易错点**：用 long 存 sum，因为路径和可能溢出 int。`map.put(0L, 1)` 处理从根开始的路径。
- **举一反三**：和为 K 的子数组（#560）→ 数组版本，不需要回溯

## 13. 二叉树的右视图（#199）

```java
public List<Integer> rightSideView(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    while (!queue.isEmpty()) {
        int size = queue.size();
        for (int i = 0; i < size; i++) {
            TreeNode node = queue.poll();
            if (i == size - 1) res.add(node.val);
            if (node.left != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
    }
    return res;
}
```

- 时间 O(n)，空间 O(n)
- 层序遍历取每层最后一个

**📝 解析**

- **思路推导**：BFS 层序遍历，每层最后一个节点就是右视图看到的。也可以 DFS 先右后左，每层第一个访问的就是右视图。
- **关键点**：`i == size - 1` 判断是否是当前层最后一个节点。
- **易错点**：DFS 写法需要记录当前深度，`depth == res.size()` 时才加入（保证每层只加一个）。
- **举一反三**：二叉树的左视图→每层第一个；二叉树的底视图→按列分组取最后一个

## 14. 二叉树的最大路径和（#124）

> 路径可以从任意节点到任意节点

```java
int maxSum = Integer.MIN_VALUE;

public int maxPathSum(TreeNode root) {
    dfs(root);
    return maxSum;
}

private int dfs(TreeNode node) {
    if (node == null) return 0;
    int left = Math.max(0, dfs(node.left));
    int right = Math.max(0, dfs(node.right));
    maxSum = Math.max(maxSum, node.val + left + right);
    return node.val + Math.max(left, right);
}
```

- 时间 O(n)，空间 O(h)
- 关键：负数贡献取 0（不选），返回值只能选一边

**📝 解析**

- **思路推导**：和直径（#543）同一个模式。对每个节点，经过它的最大路径和 = node.val + max(0,左) + max(0,右)。返回给父节点时只能选一边。
- **关键点**：`Math.max(0, dfs(...))` 负数贡献不如不选。全局变量 maxSum 初始化为 `Integer.MIN_VALUE`（可能所有节点都是负数）。
- **易错点**：返回值是 `node.val + max(left, right)` 不是 `node.val + left + right`，因为路径不能分叉。
- **举一反三**：二叉树的直径（#543）→ 同样的模式，把值换成边数

## 15. 平衡二叉树（#110）

```java
public boolean isBalanced(TreeNode root) {
    return height(root) != -1;
}

private int height(TreeNode node) {
    if (node == null) return 0;
    int l = height(node.left);
    if (l == -1) return -1;
    int r = height(node.right);
    if (r == -1) return -1;
    return Math.abs(l - r) > 1 ? -1 : 1 + Math.max(l, r);
}
```

- 时间 O(n)，空间 O(h)
- 用 -1 表示不平衡，提前剪枝

**📝 解析**

- **思路推导**：自顶向下对每个节点算高度 O(n²)。自底向上 O(n)：求高度的同时判断平衡，不平衡就返回 -1 提前终止。
- **关键点**：-1 作为"不平衡"的哨兵值，一旦子树返回 -1 就直接传播，不再计算。
- **易错点**：先检查左子树返回值，-1 就直接返回，不需要再算右子树（剪枝）。
- **举一反三**：这种"返回值复用"的技巧在树的题目中很常见，比如最大路径和也是类似思路
