# 位运算

## 核心技巧

- `n & (n - 1)`：消除最低位的 1（判断 2 的幂、数 1 的个数）
- `n ^ n = 0`，`n ^ 0 = n`：异或消消乐（找唯一出现的数）
- `n & 1`：判断奇偶（比 `n % 2` 快）
- `<<` 左移 = ×2，`>>` 右移 = ÷2
- 常用位操作：

```
获取第 k 位：    (n >> k) & 1
设置第 k 位为 1：n | (1 << k)
清除第 k 位：    n & ~(1 << k)
取最低位的 1：   n & (-n)
```

## 1. 只出现一次的数字（#136）

> 除了一个数只出现一次，其他数都出现两次，找这个数

```java
public int singleNumber(int[] nums) {
    int result = 0;
    for (int n : nums) result ^= n;
    return result;
}
```

- 时间 O(n)，空间 O(1)
- 异或：相同为 0，不同为 1

**📝 解析**

- **思路推导**：排序后相邻比较 O(n·log n)。HashMap 统计 O(n) 空间。最优：异或。`a ^ a = 0`，`a ^ 0 = a`，异或满足交换律和结合律。所有数异或后，成对的消为 0，剩下的就是唯一出现的数。
- **关键点**：异或的三个性质——①`a ^ a = 0`（自己消自己）②`a ^ 0 = a`（0 是中性元素）③交换律结合律（顺序无所谓）。这三个性质保证了结果正确。
- **易错点**：初始值是 0，不是 nums[0]。如果用 `result = nums[0]` 循环从 1 开始，少异或一个。
- **举一反三**：
  - 只出现一次的数字 II（#137）→ 出现三次，不能直接异或，需要逐位统计
  - 只出现一次的数字 III（#260）→ 两个数各出现一次，先异或再分组
  - 找不同（#389）→ 同样异或思路

## 2. 只出现一次的数字 II（#137）

> 除了一个数只出现一次，其他数都出现三次，找这个数

**思路**：逐位统计，每个 bit 位上 1 的个数 mod 3 就是答案的该位

```java
public int singleNumber(int[] nums) {
    int result = 0;
    for (int i = 0; i < 32; i++) {
        int bitCount = 0;
        for (int n : nums) {
            bitCount += (n >> i) & 1;  // 统计第 i 位有多少个 1
        }
        if (bitCount % 3 != 0) {
            result |= (1 << i);  // 第 i 位不是 3 的倍数，答案的该位是 1
        }
    }
    return result;
}
```

- 时间 O(32n) = O(n)，空间 O(1)
- 通用方法：出现 k 次就 mod k

**📝 解析**

- **思路推导**：出现三次的数，每个 bit 位贡献的 1 的个数一定是 3 的倍数。唯一出现一次的数贡献的 1 不是 3 的倍数。所以对每个 bit 位统计 1 的个数，mod 3 不为 0 就是答案的该位。
- **关键点**：这个方法通用——如果其他数出现 5 次，就 mod 5。32 位逐位处理，每位独立。
- **易错点**：负数的处理。Java 中 `>>` 是算术右移（补符号位），但这里 `(n >> i) & 1` 取最低位，对负数也正确。最后 result 会被正确解释为 int（含符号位）。
- **举一反三**：
  - 只出现一次的数字 III（#260）→ 两个数各出现一次，异或后分组
  - 这类题的本质是"频率统计"，位运算是工具

## 3. 2 的幂（#231）

> 判断 n 是否是 2 的幂

```java
public boolean isPowerOfTwo(int n) {
    return n > 0 && (n & (n - 1)) == 0;
}
```

- 时间 O(1)
- `n & (n - 1)` 消去最低位的 1

**📝 解析**

- **思路推导**：2 的幂的二进制只有 1 个 1：`1=1`、`2=10`、`4=100`、`8=1000`。`n & (n-1)` 的作用是消除最低位的 1。如果消完后为 0，说明原来只有 1 个 1，就是 2 的幂。
- **关键点**：`n > 0` 必须检查，因为 0 和负数不是 2 的幂。`n & (n-1) == 0` 只能判断是否只有 1 个 1（2 的幂），不能判断是否是 4 的幂（4 的幂要求 1 在偶数位）。
- **易错点**：`n & (n-1)` 的运算符优先级：`&` 低于 `==`，必须加括号 `(n & (n-1)) == 0`。
- **举一反三**：
  - 4 的幂（#340）→ 额外检查 `n & 0x55555555 != 0`（1 在偶数位）
  - 位 1 的个数（#191）→ 用 `n & (n-1)` 循环消 1 计数

## 4. 位 1 的个数（#191）

> 返回 n 的二进制中 1 的个数（Hamming Weight）

```java
public int hammingWeight(int n) {
    int count = 0;
    while (n != 0) {
        n &= (n - 1);  // 消去最低位的 1
        count++;
    }
    return count;
}
```

- 时间 O(1)（最多 32 次），空间 O(1)
- 比逐位检查快（只循环 1 的个数次）

**📝 解析**

- **思路推导**：逐位检查 `(n >> i) & 1` 需要循环 32 次。优化：`n & (n-1)` 每次消去最低位的 1，循环次数 = 1 的个数，通常远小于 32。
- **关键点**：`n &= (n - 1)` 等价于 `n = n & (n - 1)`，消去 n 最低位的那个 1。
- **易错点**：Java 中 n 是 int（有符号），负数的右移会补 1。但用 `n & (n-1)` 方法不受影响，因为只操作已有的位。
- **举一反三**：
  - 汉明距离（#462）→ 两个数异或后数 1 的个数
  - 比特位计数（#338）→ 对 0~n 每个数都求 1 的个数

## 5. 比特位计数（#338）

> 返回 [0, n] 每个数的二进制中 1 的个数

**思路**：DP，`dp[i] = dp[i & (i-1)] + 1`

```java
public int[] countBits(int n) {
    int[] dp = new int[n + 1];
    for (int i = 1; i <= n; i++) {
        dp[i] = dp[i & (i - 1)] + 1;
    }
    return dp;
}
```

- 时间 O(n)，空间 O(1)（不算返回数组）
- `i & (i-1)` 把最低位 1 消掉，所以 dp 值比 dp[i & (i-1)] 多 1

**📝 解析**

- **思路推导**：暴力对每个数调 hammingWeight O(n·32)。DP 优化：`i & (i-1)` 把 i 最低位的 1 消掉得到 j，那么 `dp[i] = dp[j] + 1`（i 比 j 多一个 1）。j < i，所以 dp[j] 已经算过了。
- **关键点**：状态转移方程 `dp[i] = dp[i & (i-1)] + 1`。举例：`dp[7] = dp[6] + 1`（7=111，6=110，消掉最低位 1 后是 6）。
- **易错点**：dp[0] = 0（初始值，0 有 0 个 1）。循环从 1 开始。
- **举一反三**：
  - 也可以用 `dp[i] = dp[i >> 1] + (i & 1)`（右移一位 + 最低位是否为 1）
  - 两种转移方程都对，选自己好理解的

## 6. 缺失数字（#268）

> [0, n] 中 n 个数的数组，缺了一个数，找出来

```java
public int missingNumber(int[] nums) {
    int result = nums.length;  // 注意初始值
    for (int i = 0; i < nums.length; i++) {
        result ^= i ^ nums[i];
    }
    return result;
}
```

- 时间 O(n)，空间 O(1)
- 异或：下标和值一起异或，成对消除，剩下的就是缺失的

**📝 解析**

- **思路推导**：把下标 0~n 和数组值一起异或。下标和值如果匹配就成对消除（`a ^ a = 0`），缺失的那个数只出现一次（在下标中出现但不在值中出现），最后剩下的就是它。
- **关键点**：初始值是 `nums.length`（即 n），因为下标是 0~n-1，但完整范围是 0~n，n 只在下标侧出现。或者换个理解：把 n 也纳入异或，等价于"下标 0~n 异或 值 0~n-1"。
- **易错点**：初始值不是 0，是 `nums.length`。如果用 `result = 0` 循环从 0 到 n-1，会漏掉 n。
- **举一反三**：寻找消失的数字（#448）→ [1,n] 的数组找缺失的多个数，用原地标记法（把对应下标取负）

## 位运算总结

```
┌───────────────────────────────────────────────────┐
│              位运算核心公式                          │
├───────────────────────────────────────────────────┤
│ n & (n - 1)    → 消去最低位的 1                     │
│ n & (-n)       → 取最低位的 1（lowbit）             │
│ n ^ n = 0      → 异或消消乐                         │
│ n & 1          → 判断奇偶                           │
│ n << 1         → ×2                                │
│ n >> 1         → ÷2                                │
├───────────────────────────────────────────────────┤
│ 高频应用：                                         │
│ - 找唯一/缺失数 → 异或                              │
│ - 判断 2 的幂   → n & (n-1) == 0                  │
│ - 数 1 的个数   → 循环 n &= (n-1)                  │
│ - 逐位统计      → (n >> i) & 1                     │
└───────────────────────────────────────────────────┘
```
