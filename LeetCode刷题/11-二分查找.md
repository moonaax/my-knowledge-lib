# 二分查找

## 核心技巧

- 二分本质：**排除一半**，不是"找目标"
- 两种模板：
  - **左闭右闭** `[left, right]`：`while (left <= right)`，`right = mid - 1`
  - **左闭右开** `[left, right)`：`while (left < right)`，`right = mid`
- 三个变体：找目标值、找左边界（第一个 ≥ target）、找右边界（最后一个 ≤ target）
- **二分答案**：答案有单调性时，对答案范围二分，用 check 函数验证

---

## 1. 二分查找（#704）

> 在升序数组中查找目标值

```java
public int search(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;  // 防溢出
        if (nums[mid] == target) return mid;
        else if (nums[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}
```

- 时间 O(log n)，空间 O(1)
- `left + (right - left) / 2` 防止 `left + right` 溢出

**📝 解析**

- **思路推导**：有序数组 → 每次比较排除一半 → O(log n)。`mid` 命中就返回，比 target 小说明在右半边（left = mid+1），比 target 大说明在左半边（right = mid-1）。
- **关键点**：`left + (right - left) / 2` 不会溢出，`（left + right）/ 2` 在 left 和 right 都很大时会溢出 int。
- **易错点**：`left <= right` 有等号，因为 `[left, right]` 是闭区间，left==right 时还要检查。如果写成 `left < right` 会漏掉一个元素。
- **举一反三**：这是最基础的模板，下面的变体都基于这个框架修改。

## 2. 搜索插入位置（#35）

> 找 target 的位置，不存在则返回应该插入的位置

**思路**：找第一个 ≥ target 的位置（左边界）

```java
public int searchInsert(int[] nums, int target) {
    int left = 0, right = nums.length;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] < target) left = mid + 1;
        else right = mid;
    }
    return left;
}
```

- 时间 O(log n)
- 这就是 Java `Arrays.binarySearch` 的插入点逻辑

**📝 解析**

- **思路推导**：等价于"找第一个 ≥ target 的位置"。用左闭右开模板 `[left, right)`，`nums[mid] < target` 时 left = mid+1，否则 right = mid。循环结束时 left = right = 插入位置。
- **关键点**：为什么用左闭右开？因为答案范围是 `[0, nums.length]`（可能插在末尾），右边界取 length，开区间更自然。
- **易错点**：和基础二分的区别在于：找到相等的也不返回，而是继续收缩到左边界。这保证了返回的是"第一个"满足条件的位置。
- **举一反三**：
  - 在排序数组中查找元素的第一个和最后一个位置（#34）→ 左右边界各找一次
  - 第一个错误的版本（#278）→ 同样的左边界模板

## 3. 在排序数组中查找元素的第一个和最后一个位置（#34）

> 找 target 的起始和结束位置，不存在返回 [-1, -1]

**思路**：分别找左边界和右边界

```java
public int[] searchRange(int[] nums, int target) {
    int first = findBound(nums, target, true);
    if (first == -1) return new int[]{-1, -1};
    int last = findBound(nums, target, false);
    return new int[]{first, last};
}

private int findBound(int[] nums, int target, boolean isFirst) {
    int left = 0, right = nums.length - 1, result = -1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) {
            result = mid;
            if (isFirst) right = mid - 1;  // 继续往左找
            else left = mid + 1;           // 继续往右找
        } else if (nums[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return result;
}
```

- 时间 O(log n)，两次二分
- 一个函数复用，通过 `isFirst` 参数区分方向

**📝 解析**

- **思路推导**：普通二分找到 target 就返回。这里需要找"第一个"和"最后一个"——找到 target 后不急着返回，而是记录位置继续往左/右搜索。
- **关键点**：找到 target 后，找左边界就 `right = mid - 1`（继续往左），找右边界就 `left = mid + 1`（继续往右）。result 记录最后找到的位置。
- **易错点**：找不到 target 时返回 -1，不是返回插入位置。先找左边界，如果左边界是 -1 就不用找右边界了。
- **举一反三**：这个"找到不返回，继续搜索"的思想是所有边界搜索的核心。

## 4. 搜索旋转排序数组（#33）

> 升序数组在某个位置旋转了（如 `[4,5,6,7,0,1,2]`），搜索 target

**思路**：二分时判断哪半边是有序的

```java
public int search(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) return mid;

        // 判断哪半边有序
        if (nums[left] <= nums[mid]) {
            // 左半边有序
            if (nums[left] <= target && target < nums[mid]) right = mid - 1;
            else left = mid + 1;
        } else {
            // 右半边有序
            if (nums[mid] < target && target <= nums[right]) left = mid + 1;
            else right = mid - 1;
        }
    }
    return -1;
}
```

- 时间 O(log n)
- 关键：至少有一半是有序的

**📝 解析**

- **思路推导**：旋转后数组不是全局有序，但二分后至少一半是有序的。判断条件：`nums[left] <= nums[mid]` → 左半边有序。在有序的那半边判断 target 是否在范围内，在就搜索那半边，否则搜另一半。
- **关键点**：`nums[left] <= nums[mid]` 有等号（left == mid 时左半边只有一个元素，也算有序）。判断 target 是否在有序范围内用 `nums[left] <= target && target < nums[mid]`（左闭右开）。
- **易错点**：`<=` 和 `<` 的边界容易搞混。记住：有序半边用闭区间判断（两端都包含），然后搜索另一半。
- **举一反三**：搜索旋转排序数组 II（#81）→ 有重复元素，最坏 O(n)，需要额外处理 `nums[left] == nums[mid] == nums[right]` 的情况

## 5. 寻找旋转排序数组中的最小值（#153）

> 旋转后的升序数组找最小值

**思路**：二分，比较 nums[mid] 和 nums[right]

```java
public int findMin(int[] nums) {
    int left = 0, right = nums.length - 1;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] > nums[right]) {
            left = mid + 1;  // 最小值在右半边
        } else {
            right = mid;     // 最小值在左半边（含 mid）
        }
    }
    return nums[left];
}
```

- 时间 O(log n)
- 用左闭右开模板，`right = mid` 而不是 `mid - 1`

**📝 解析**

- **思路推导**：旋转数组的最小值就是"断崖"的位置。`nums[mid] > nums[right]` 说明最小值在 mid 右边（因为左边有序且都比右边大）。否则最小值在 mid 或 mid 左边。
- **关键点**：和 `nums[right]` 比而不是和 `nums[left]` 比。因为 `nums[mid] > nums[left]` 不能确定最小值在哪（可能在左边有序部分，也可能在右边）。
- **易错点**：`right = mid` 不是 `mid - 1`，因为 mid 可能就是最小值。用 `left < right`（无等号），循环结束时 left == right 就是最小值。
- **举一反三**：寻找旋转排序数组中的最小值 II（#154）→ 有重复元素，`nums[mid] == nums[right]` 时 right--（无法判断，只能缩小范围）

## 6. 分割数组的最大值 / 切蛋糕（#410）

> 把数组分成 K 个连续子数组，使每个子数组和的最大值最小

**思路**：二分答案，对"最大子数组和"二分

```java
public int splitArray(int[] nums, int k) {
    int left = 0, right = 0;
    for (int n : nums) {
        left = Math.max(left, n);  // 至少要能容纳最大元素
        right += n;                // 至多是全部元素之和
    }

    while (left < right) {
        int mid = left + (right - left) / 2;
        if (canSplit(nums, k, mid)) right = mid;  // 能分，尝试更小
        else left = mid + 1;                       // 不能分，需要更大
    }
    return left;
}

private boolean canSplit(int[] nums, int k, int maxSum) {
    int count = 1, currSum = 0;
    for (int n : nums) {
        if (currSum + n > maxSum) {
            count++;
            currSum = n;
        } else {
            currSum += n;
        }
    }
    return count <= k;
}
```

- 时间 O(n·log(sum))，sum 为数组总和
- **二分答案**经典题

**📝 解析**

- **思路推导**：直接枚举所有分法太复杂。换个角度——"最大子数组和"有一个范围 `[max(nums), sum(nums)]`。这个范围有**单调性**：maxSum 越大，能分出的子数组越少。所以可以对 maxSum 二分。
- **关键点**：`canSplit` 函数贪心地分配——当前子数组加上下一个数不超过 maxSum 就继续加，否则开新子数组。如果需要的子数组数 ≤ k，说明 maxSum 可行。
- **易错点**：left 初始是 `max(nums)` 不是 0（最大子数组和至少要能容纳最大的单个元素）。`count <= k` 不是 `== k`（分得更少也行，说明 maxSum 有冗余）。
- **举一反三**：
  - 爱吃香蕉的珂珂（#875）→ 二分吃香蕉的速度
  - 在 D 天内送达包裹的能力（#1011）→ 二分运载能力
  - 这类题统称"**二分答案**"：答案有单调性 + 能用贪心验证 → 二分答案范围

## 二分查找总结

```
┌──────────────────────────────────────────────────┐
│              二分查找的三种模式                      │
├──────────────────────────────────────────────────┤
│ 1. 找目标值    → 找到就返回                         │
│ 2. 找边界      → 找到不返回，记录后继续收缩           │
│ 3. 二分答案    → 对答案范围二分，用 check 验证        │
├──────────────────────────────────────────────────┤
│ 关键判断：                                        │
│ - left <= right（闭区间）还是 left < right（开区间）│
│ - 找到后 left = mid+1 还是 right = mid-1          │
│ - 答案是否具有单调性（二分答案的前提）                │
└──────────────────────────────────────────────────┘
```
