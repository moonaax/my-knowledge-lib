# 并查集

## 核心技巧

- 并查集（Union-Find）：高效处理**连通性**问题——判断两个元素是否在同一集合，合并两个集合
- 两大优化：**路径压缩**（find 时扁平化树）+ **按秩合并**（合并时矮树接高树）
- 优化后 find/union 均为 O(α(n)) ≈ O(1)，α 是反阿克曼函数
- 适用场景：连通分量计数、检测环、动态连通性

---

## 并查集模板

```java
class UnionFind {
    private int[] parent;
    private int[] rank;
    private int count;  // 连通分量数

    public UnionFind(int n) {
        parent = new int[n];
        rank = new int[n];
        count = n;
        for (int i = 0; i < n; i++) parent[i] = i;  // 初始时每个节点是自己的父节点
    }

    // 查找根节点（带路径压缩）
    public int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);  // 路径压缩：直接指向根
        }
        return parent[x];
    }

    // 合并两个集合（按秩合并）
    public boolean union(int x, int y) {
        int rootX = find(x), rootY = find(y);
        if (rootX == rootY) return false;  // 已经在同一集合

        if (rank[rootX] < rank[rootY]) {
            parent[rootX] = rootY;
        } else if (rank[rootX] > rank[rootY]) {
            parent[rootY] = rootX;
        } else {
            parent[rootY] = rootX;
            rank[rootX]++;
        }
        count--;
        return true;
    }

    public boolean connected(int x, int y) {
        return find(x) == find(y);
    }

    public int getCount() { return count; }
}
```

**模板要点**：

| 操作 | 作用 | 复杂度 |
|------|------|--------|
| `find(x)` | 找 x 的根节点，路径压缩 | O(α(n)) |
| `union(x, y)` | 合并两个集合，返回是否新合并 | O(α(n)) |
| `connected(x, y)` | 判断是否连通 | O(α(n)) |
| `getCount()` | 当前连通分量数 | O(1) |

## 1. 省份数量（#547）

> n 个城市，isConnected[i][j]=1 表示 i 和 j 直连，求连通分量数

**思路**：并查集，遍历邻接矩阵合并连通的城市

```java
public int findCircleNum(int[][] isConnected) {
    int n = isConnected.length;
    UnionFind uf = new UnionFind(n);
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            if (isConnected[i][j] == 1) uf.union(i, j);
        }
    }
    return uf.getCount();
}
```

- 时间 O(n²·α(n))，空间 O(n)
- 直接用模板

**📝 解析**

- **思路推导**：城市是节点，直连是边，求连通分量数 → 并查集的经典应用。遍历上三角邻接矩阵，有连接就 union，最后返回 count。
- **关键点**：`j = i + 1` 只遍历上三角（对称矩阵，避免重复 union）。`union` 返回 false 说明已经在同一集合，count 不会重复减。
- **易错点**：也可以用 DFS/BFS 求连通分量，并查集的优势在于支持动态添加边（在线算法）。
- **举一反三**：
  - 被围绕的区域（#130）→ 虽然在图专题用 DFS 做了，但并查集也能做
  - 连通网络的操作次数（#1319）→ 连通分量数 - 1 就是需要的连接数

## 2. 冗余连接（#684）

> 树多了一条边变成环，找出这条多余的边

**思路**：逐条加边，union 返回 false 说明这条边造成了环

```java
public int[] findRedundantConnection(int[][] edges) {
    int n = edges.length;
    UnionFind uf = new UnionFind(n + 1);  // 节点编号从 1 开始
    for (int[] edge : edges) {
        if (!uf.union(edge[0], edge[1])) {
            return edge;  // union 失败 → 两端已连通 → 这条边是多余的
        }
    }
    return new int[0];
}
```

- 时间 O(n·α(n))
- union 返回 false 的那条边就是答案

**📝 解析**

- **思路推导**：一棵树有 n-1 条边，n 个节点。多了一条边就一定有环。逐条加边，如果一条边的两个端点已经连通（find 返回相同根），再加这条边就会形成环——这就是冗余边。
- **关键点**：`union` 返回 false 等价于"两个端点已经连通"。最后出现的冗余边就是答案（题目保证唯一解时取最后一条）。
- **易错点**：节点编号从 1 开始，所以 UnionFind 大小是 `n + 1`。
- **举一反三**：
  - 冗余连接 II（#685）→ 有向图，可能是多一个入度的节点，需要分类讨论，困难题
  - 用 DFS 检测环也行，但并查集更简洁

## 3. 账户合并（#721）

> 多个账户可能有相同邮箱，合并同一人的所有邮箱

**思路**：同一账户内的邮箱互相 union，最后按根节点分组

```java
public List<List<String>> accountsMerge(List<List<String>> accounts) {
    // email → index 的映射
    Map<String, Integer> emailToIdx = new HashMap<>();
    UnionFind uf = new UnionFind(accounts.size() * 10);  // 足够大

    // 同一账户内的邮箱互相 union
    for (int i = 0; i < accounts.size(); i++) {
        for (int j = 1; j < accounts.get(i).size(); j++) {
            String email = accounts.get(i).get(j);
            if (emailToIdx.containsKey(email)) {
                uf.union(emailToIdx.get(email), i * 10);
            } else {
                emailToIdx.put(email, i * 10);
            }
        }
    }

    // 按根节点分组
    Map<Integer, TreeSet<String>> groups = new HashMap<>();
    for (Map.Entry<String, Integer> e : emailToIdx.entrySet()) {
        int root = uf.find(e.getValue());
        groups.computeIfAbsent(root, k -> new TreeSet<>()).add(e.getKey());
    }

    // 构造结果
    List<List<String>> result = new ArrayList<>();
    for (TreeSet<String> group : groups.values()) {
        List<String> merged = new ArrayList<>();
        // 第一个元素是账户名（通过 root 反推）
        String name = accounts.get(group.first().equals("") ? 0 :
            emailToIdx.get(group.first()) / 10).get(0);
        merged.add(name);
        merged.addAll(group);
        result.add(merged);
    }
    return result;
}
```

- 时间 O(N·α(N))，N 为总邮箱数
- 用 TreeSet 自动排序

**📝 解析**

- **思路推导**：关键观察——如果两个账户有相同邮箱，它们属于同一个人。所以同一账户内的邮箱 union 在一起，不同账户如果共享邮箱也 union 在一起。最后每个连通分量就是一个合并后的账户。
- **关键点**：用 `emailToIdx` 把邮箱映射到编号，同一个邮箱在不同账户中出现时 union。用 `TreeSet` 自动去重排序。
- **易错点**：UnionFind 大小要足够。账户名可以通过 root 反推（记录每个 root 对应的账户名）。实际实现中可以把 `i * 10` 换成更精确的映射。
- **举一反三**：等式方程的可满足性（#990）→ 同样的思路，`==` 就 union，`!=` 就检查是否连通

## 4. 等式方程的可满足性（#990）

> 给定 `["a==b","b!=c","c==a"]`，判断是否矛盾

**思路**：先 union 所有 `==`，再检查所有 `!=` 是否矛盾

```java
public boolean equationsPossible(String[] equations) {
    UnionFind uf = new UnionFind(26);
    // 第一遍：处理 == 方程
    for (String eq : equations) {
        if (eq.charAt(1) == '=') {
            uf.union(eq.charAt(0) - 'a', eq.charAt(3) - 'a');
        }
    }
    // 第二遍：检查 != 方程
    for (String eq : equations) {
        if (eq.charAt(1) == '!') {
            if (uf.connected(eq.charAt(0) - 'a', eq.charAt(3) - 'a')) {
                return false;  // 矛盾：不相等但实际连通
            }
        }
    }
    return true;
}
```

- 时间 O(n·α(26)) ≈ O(n)
- 字符映射到 0-25

**📝 解析**

- **思路推导**：`==` 表示两个变量在同一集合，`!=` 表示不在同一集合。先用并查集合并所有 `==` 的变量，然后检查每个 `!=` 的两个变量是否意外连通了。
- **关键点**：两遍扫描——先处理所有 `==`（建立连通关系），再检查所有 `!=`（验证是否矛盾）。顺序不能反。
- **易错点**：`eq.charAt(1)` 判断是 `=` 还是 `!`（字符串格式固定为 `a==b` 或 `a!=b`）。字符转下标用 `- 'a'`。
- **举一反三**：这道题是并查集的经典应用模式——**先建立等价关系，再验证约束条件**。

│ 模板关键：                                           │
│ - find 带路径压缩（递归直接指向根）                     │
│ - union 按秩合并（矮树接高树）                         │
