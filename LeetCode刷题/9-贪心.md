# 贪心

## 核心技巧

- 贪心本质：每一步选局部最优，期望全局最优
- 难点在于证明贪心策略的正确性
- 常见场景：区间调度、跳跃游戏、分配问题
- 想不出贪心就用 DP，贪心是 DP 的特殊情况

---

## 1. 跳跃游戏（#55）

> 判断能否从起点跳到终点

```java
public boolean canJump(int[] nums) {
    int maxReach = 0;
    for (int i = 0; i < nums.length; i++) {
        if (i > maxReach) return false;
        maxReach = Math.max(maxReach, i + nums[i]);
    }
    return true;
}
```

- 维护能到达的最远位置
- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：贪心：遍历时维护能到达的最远位置 maxReach。如果当前位置 i > maxReach，说明到不了这里，返回 false。
- **关键点**：不需要真的跳，只需要知道"能不能到"。每个位置更新 maxReach = max(maxReach, i+nums[i])。
- **易错点**：`i > maxReach` 要在更新之前判断，否则已经到不了的位置还在更新。
- **举一反三**：跳跃游戏 II（#45）→ 求最少跳跃次数，贪心策略不同

## 2. 跳跃游戏 II（#45）

> 最少跳几次到终点

```java
public int jump(int[] nums) {
    int jumps = 0, curEnd = 0, farthest = 0;
    for (int i = 0; i < nums.length - 1; i++) {
        farthest = Math.max(farthest, i + nums[i]);
        if (i == curEnd) {
            jumps++;
            curEnd = farthest;
        }
    }
    return jumps;
}
```

- 贪心：每次跳到能到达最远的位置
- 时间 O(n)，空间 O(1)

**📝 解析**

- **思路推导**：BFS 思想：curEnd 是当前跳跃的边界，farthest 是下一跳能到的最远位置。到达 curEnd 时必须跳一次，更新 curEnd = farthest。
- **关键点**：遍历到 `nums.length - 1` 之前（不包括最后一个），因为到达最后一个不需要再跳。
- **易错点**：`i == curEnd` 时 jumps++，不是 `i == farthest`。curEnd 是"必须跳"的触发点。
- **举一反三**：跳跃游戏（#55）→ 简化版，只判断能否到达

## 3. 无重叠区间（#435）

> 移除最少区间使剩余区间不重叠

**思路**：按右端点排序，贪心选结束最早的

```java
public int eraseOverlapIntervals(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[1] - b[1]);
    int count = 0, end = Integer.MIN_VALUE;
    for (int[] interval : intervals) {
        if (interval[0] >= end) {
            end = interval[1];
        } else {
            count++;
        }
    }
    return count;
}
```

- 时间 O(n·log n)，空间 O(1)
- 经典区间调度问题

**📝 解析**

- **思路推导**：等价于"最多保留多少个不重叠区间"，用总数减去保留数就是移除数。按右端点排序，贪心选结束最早的，给后面留更多空间。
- **关键点**：按右端点排序而不是左端点。选结束早的 = 给后续区间留最大空间 = 保留最多区间。
- **易错点**：`interval[0] >= end` 是不重叠（等于时不算重叠），不是 `>`。
- **举一反三**：合并区间（#56）→ 按左端点排序合并；用最少数量的箭引爆气球（#452）→ 几乎一样

## 4. 分发糖果（#135）

> 每个孩子至少一颗糖，评分高的比邻居多

**思路**：两次遍历，左到右 + 右到左

```java
public int candy(int[] ratings) {
    int n = ratings.length;
    int[] candies = new int[n];
    Arrays.fill(candies, 1);
    for (int i = 1; i < n; i++)
        if (ratings[i] > ratings[i - 1]) candies[i] = candies[i - 1] + 1;
    for (int i = n - 2; i >= 0; i--)
        if (ratings[i] > ratings[i + 1]) candies[i] = Math.max(candies[i], candies[i + 1] + 1);
    int sum = 0;
    for (int c : candies) sum += c;
    return sum;
}
```

- 时间 O(n)，空间 O(n)

**📝 解析**

- **思路推导**：一次遍历无法同时满足左右邻居约束。两次遍历：左到右保证右边比左边高的多给，右到左保证左边比右边高的多给，取 max。
- **关键点**：第二遍取 `max(candies[i], candies[i+1]+1)`，不能直接赋值，因为要保留第一遍的结果。
- **易错点**：相邻评分相同时不要求糖果数不同，所以相等时不处理。
- **举一反三**：这种"两次遍历取 max"的贪心模式在接雨水的预处理解法中也有体现

## 5. 加油站（#134）

> 环形路线，找能跑完一圈的起点

```java
public int canCompleteCircuit(int[] gas, int[] cost) {
    int total = 0, tank = 0, start = 0;
    for (int i = 0; i < gas.length; i++) {
        int diff = gas[i] - cost[i];
        total += diff;
        tank += diff;
        if (tank < 0) {
            start = i + 1;
            tank = 0;
        }
    }
    return total >= 0 ? start : -1;
}
```

- 时间 O(n)，空间 O(1)
- 贪心：如果从 i 到不了 j，那 i 到 j 之间的任何点都到不了 j

**📝 解析**

- **思路推导**：暴力从每个站出发模拟 O(n²)。贪心：如果从 i 出发到 j 油量为负，那 i~j 之间任何站出发都到不了 j（因为中间的累积油量更少），直接跳到 j+1 重新开始。
- **关键点**：`total >= 0` 说明一定有解（题目保证唯一解时）。tank 记录当前段的油量，total 记录全局油量。
- **易错点**：`start = i + 1` 而不是 `start = i`，当前站已经证明不行了。
- **举一反三**：这个"失败就跳过整段"的贪心思想在很多环形问题中都有应用
